# Лекция: Алгоритм Кнута-Морриса-Пратта

## 1. Введение

**Алгоритм Кнута-Морриса-Пратта (КМП)** — эффективный алгоритм поиска подстроки в строке, использующий информацию о структуре образца для избежания лишних сравнений.

Алгоритм был разработан Дональдом Кнутом и Воном Праттом, а также независимо Джеймсом Моррисом. Результаты опубликованы совместно в 1977 году.

**Ключевая идея:** при несовпадении символов образец содержит достаточно информации, чтобы определить, где может начаться следующее совпадение.

**Сложность:** `O(n + m)` — линейная, асимптотически оптимальная.

---

## 2. Подход на основе конечного автомата

### 2.1 Построение автомата

Конечный автомат для поиска строится по образцу. Состояния автомата соответствуют префиксам образца.

Для образца `ABABC`:
- Состояния: {∅, A, AB, ABA, ABAB, ABABC}
- Алфавит: {A, B, C}
- Начальное состояние: ∅
- Конечное состояние: ABABC

### 2.2 Таблица переходов

Таблица `δ[state][char]` определяет следующее состояние при чтении символа.

Для `ABABC`:

| Состояние | A | B | C |
|-----------|---|---|---|
| 0 (∅)     | 1 | 0 | 0 |
| 1 (A)     | 1 | 2 | 0 |
| 2 (AB)    | 3 | 0 | 0 |
| 3 (ABA)   | 1 | 4 | 0 |
| 4 (ABAB)  | 3 | 0 | 5 |
| 5 (ABABC) | — | — | — |

### 2.3 Построение таблицы переходов

```cpp
// Для каждого состояния i и символа c
// Находим максимальный k такой, что:
// prefix(pattern, k) == suffix(prefix(pattern, i) + c)

void buildDelta() {
    for (int state = 0; state <= m; state++) {
        for (char c : alphabet) {
            string line = pattern.substr(0, state) + c;
            int k = min(state + 1, m);
            
            while (k > 0 && pattern.substr(0, k) != suffix(line, k)) {
                k--;
            }
            
            delta[state][c] = k;
        }
    }
}
```

**Сложность построения:** O(m³ · |Σ|) — из-за сравнения строк в цикле.

### 2.4 Поиск с использованием автомата

```cpp
vector<int> search(const string& text) {
    vector<int> results;
    int state = 0;
    
    for (int i = 0; i < text.length(); i++) {
        state = delta[state][text[i]];
        
        if (state == m) {
            results.push_back(i - m + 1);
        }
    }
    
    return results;
}
```

**Сложность поиска:** O(n) — один проход по тексту.

## 3. Префикс-функция

### 3.1 Определение

**Префикс-функция π[i]** — длина наибольшего собственного суффикса строки `pattern[0..i]`, который одновременно является её префиксом.

Иначе говоря: максимальная длина k < i+1, такая что:
```
pattern[0..k-1] == pattern[i-k+1..i]
```

### 3.2 Пример

Для `ABABC`:

| i | pattern[0..i] | π[i] | Объяснение |
|---|---------------|------|------------|
| 0 | A             | 0    | Нет собственного префикса = суффикса |
| 1 | AB            | 0    | A ≠ B |
| 2 | ABA           | 1    | A = A (последний символ = первый) |
| 3 | ABAB          | 2    | AB = AB |
| 4 | ABABC         | 0    | Нет совпадений |

π = [0, 0, 1, 2, 0]

### 3.3 Медленный алгоритм вычисления π-функции

Наивный подход — для каждой позиции проверяем все возможные длины:

```cpp
vector<int> computePrefixSlow(const string& pattern) {
    int m = pattern.length();
    vector<int> pi(m, 0);
    
    for (int i = 1; i < m; i++) {
        // Перебираем все возможные длины k от i до 1
        for (int k = i; k > 0; k--) {
            // Проверяем: prefix[0..k-1] == suffix[i-k+1..i]
            bool match = true;
            for (int j = 0; j < k; j++) {
                if (pattern[j] != pattern[i - k + 1 + j]) {
                    match = false;
                    break;
                }
            }
            if (match) {
                pi[i] = k;
                break;
            }
        }
    }
    
    return pi;
}
```

**Сложность:** O(m³) — три вложенных цикла.

### 3.4 Быстрый алгоритм вычисления π-функции

Ключевое наблюдение: если π[i] = k, то при вычислении π[i+1] мы можем использовать уже вычисленные значения.

```cpp
vector<int> computePrefixFast(const string& pattern) {
    int m = pattern.length();
    vector<int> pi(m, 0);
    
    for (int i = 1; i < m; i++) {
        int len = pi[i - 1];  // Длина предыдущего совпадения
        
        // Пока не нашли совпадение и есть куда отступать
        while (len > 0 && pattern[len] != pattern[i]) {
            len = pi[len - 1];  // Откатываемся по π-функции
        }
        
        // Если символы совпали, увеличиваем длину
        if (pattern[len] == pattern[i]) {
            len++;
        }
        
        pi[i] = len;
    }
    
    return pi;
}
```

**Сложность:** O(m) — амортизированная линейная.

**Почему O(m)?** 
- Каждое увеличение `len` соответствует продвижению по строке
- Каждый откат `len = pi[len-1]` уменьшает `len`
- Суммарно откатов не больше, чем увеличений
- Увеличений не более m → откатов не более m

## 4. Алгоритм КМП

### 4.1 Основная идея

Вместо построения полного автомата используем только π-функцию. При несовпадении символа `text[i]` с `pattern[j]` откатываемся к позиции `j = π[j-1]`.

### 4.2 Метод конкатенации

Простой способ реализации — склеить образец и текст через разделитель:

```
pattern + "#" + text
```

Где `#` — символ, не встречающийся ни в образце, ни в тексте.

Затем вычисляем π-функцию для всей строки. Если `π[i] == m`, значит нашли вхождение.

```cpp
vector<int> searchKMP(const string& text, const string& pattern) {
    string combined = pattern + "#" + text;
    vector<int> pi = computePrefixFast(combined);
    
    vector<int> results;
    int m = pattern.length();
    
    for (int i = m + 1; i < combined.length(); i++) {
        if (pi[i] == m) {
            // Позиция в исходном тексте
            results.push_back(i - 2 * m);
        }
    }
    
    return results;
}
```

### 4.3 Оптимизированная реализация (без конкатенации)

Более эффективная версия не создаёт новую строку:

```cpp
vector<int> searchKMPOptimized(const string& text, const string& pattern) {
    int n = text.length();
    int m = pattern.length();
    
    vector<int> pi = computePrefixFast(pattern);
    vector<int> results;
    
    int j = 0;  // Текущая позиция в образце
    
    for (int i = 0; i < n; i++) {
        // Откатываемся пока не найдём совпадение или j = 0
        while (j > 0 && pattern[j] != text[i]) {
            j = pi[j - 1];
        }
        
        // Если символы совпали
        if (pattern[j] == text[i]) {
            j++;
        }
        
        // Если дошли до конца образца — нашли вхождение
        if (j == m) {
            results.push_back(i - m + 1);
            j = pi[j - 1];  // Продолжаем поиск
        }
    }
    
    return results;
}
```

## 5. Сравнение подходов

| Характеристика | Конечный автомат | КМП |
|----------------|------------------|-----|
| Препроцессинг | O(m³ · \|Σ\|) или O(m · \|Σ\|) | O(m) |
| Память | O(m · \|Σ\|) | O(m) |
| Поиск | O(n) | O(n) |
| Общая сложность | O(m · \|Σ\| + n) | O(m + n) |

**Когда использовать автомат:**
- Большой алфавит не критичен
- Многократный поиск одного образца
- Нужна максимальная скорость поиска

**Когда использовать КМП:**
- Ограничения по памяти
- Однократный поиск
- Большой алфавит

---

## 6. Сравнение с алгоритмом Бойера-Мура

| Характеристика | КМП | Бойер-Мур |
|----------------|-----|-----------|
| Направление сравнения | Слева направо | Справа налево |
| Лучший случай | O(n) | O(n/m) |
| Худший случай | O(n + m) | O(n + m) |
| Практическая скорость | Стабильная | Быстрее на естественных текстах |
| Память | O(m) | O(m + \|Σ\|) |

**КМП лучше когда:**
- Важна гарантированная линейная сложность
- Алфавит маленький (ДНК, бинарные данные)
- Образец содержит много повторений

**Бойер-Мур лучше когда:**
- Большой алфавит (естественный язык)
- Образец и текст сильно различаются

## 9. Итоги

- **Конечный автомат** — явное построение таблицы переходов, O(n) поиск
- **Префикс-функция π[i]** — длина максимального собственного префикса-суффикса
- **Медленное вычисление π** — O(m³), наивный перебор
- **Быстрое вычисление π** — O(m), использование предыдущих значений
- **КМП** — O(n + m), оптимальная асимптотика для поиска подстроки

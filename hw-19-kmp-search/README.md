## Реализованные алгоритмы

### 1. AutomatonSearch — Конечный автомат (2 балла)

- **Препроцессинг:** O(m² · |Σ|) — построение таблицы переходов
- **Поиск:** O(n) — один проход по тексту
- **Память:** O(m · |Σ|)

Строит явную таблицу переходов `δ[state][char]` для образца.

### 2. KMPSearchSlow — Медленная π-функция (2 балла)

- **Препроцессинг:** O(m³) — наивное вычисление π-функции
- **Поиск:** O(n)
- **Память:** O(m)

Для каждой позиции перебирает все возможные длины префикса-суффикса.

### 3. KMPSearch — Быстрая π-функция (2 балла)

- **Препроцессинг:** O(m) — эффективное вычисление π-функции
- **Поиск:** O(n)
- **Общая сложность:** O(n + m) — оптимальная
- **Память:** O(m)

Использует ранее вычисленные значения для "отката" по π-функции.

## Пример π-функции

Для образца `ABABC`:

| i | pattern[0..i] | π[i] | Объяснение |
|---|---------------|------|------------|
| 0 | A             | 0    | Нет собственного префикса = суффикса |
| 1 | AB            | 0    | A ≠ B |
| 2 | ABA           | 1    | A = A |
| 3 | ABAB          | 2    | AB = AB |
| 4 | ABABC         | 0    | Нет совпадений |

## Выводы

1. **AutomatonSearch** — быстрый поиск O(n), но медленное построение O(m²·|Σ|).
   Эффективен при многократном поиске одного образца.

2. **KMPSearchSlow** — демонстрирует наивный подход O(m³).
   Практически не применим для длинных образцов.

3. **KMPSearch** — оптимальный алгоритм O(n+m).
   Гарантированная линейная сложность, минимум памяти.

**Сравнение с Boyer-Moore:**
- КМП гарантирует линейность в худшем случае
- Boyer-Moore быстрее на естественных текстах (O(n/m) в лучшем случае)
- КМП лучше для маленького алфавита (ДНК, бинарные данные)

## API использования

```cpp
#include "search/KMPSearch.hpp"

// Создание поисковика
KMPSearch searcher;

// Установка образца (вычисляет π-функцию)
searcher.setPattern("ABABC");

// Поиск первого вхождения
int pos = searcher.search("ABABABCCBBABABCAB");  // → 2

// Поиск всех вхождений
auto all = searcher.searchAll("ABABABCCBBABABCAB");  // → [2, 10]

// Доступ к π-функции
const auto& pi = searcher.getPrefixFunction();  // → [0, 0, 1, 2, 0]
```

# ДЗ-16: Минимальное остовное дерево (MST)

Реализация алгоритмов построения минимального остовного дерева с использованием собственных структур данных из ДЗ-04 (arrayslib).

## Реализованные алгоритмы

### 1. Алгоритм Краскала (основное задание)

**Назначение:** построение MST жадным выбором рёбер минимального веса.

**Идея алгоритма:**
1. Сортируем все рёбра по весу
2. Проходим по рёбрам в порядке возрастания веса
3. Добавляем ребро в MST, если оно не образует цикл
4. Для проверки циклов используется структура Union-Find

```cpp
FactorArray<WeightedEdge> edges;
edges.add(WeightedEdge(0, 1, 4), edges.size());
edges.add(WeightedEdge(0, 2, 3), edges.size());
edges.add(WeightedEdge(1, 2, 1), edges.size());
edges.add(WeightedEdge(1, 3, 2), edges.size());

MSTResult result = KruskalMST::execute(edges, 4);
// MST: {1-2, 1-3, 0-2}, вес = 6
```

**Сложность:** O(E log E) — определяется сортировкой рёбер

---

### 2. Алгоритм Прима (дополнительное задание 1)

**Назначение:** построение MST "выращиванием" дерева из начальной вершины.

**Идея алгоритма:**
1. Начинаем с вершины 0
2. На каждом шаге добавляем минимальное ребро, ведущее из дерева наружу
3. Повторяем, пока все вершины не будут включены

```cpp
SpaceArray<int> adjList;  // Список смежности с весами
// adjList.get(v, 2*i) = сосед, adjList.get(v, 2*i+1) = вес

MSTResult result = PrimMST::execute(adjList, vertexCount);
```

**Сложность:** O(V²) — данная реализация с массивом

---

### 3. Алгоритм Борувки (дополнительное задание 2)

**Назначение:** построение MST параллельным объединением компонент.

**Идея алгоритма:**
1. Изначально каждая вершина — отдельная компонента
2. Для каждой компоненты находим минимальное ребро наружу
3. Добавляем все найденные рёбра в MST
4. Повторяем, пока не останется одна компонента

```cpp
MSTResult result = BoruvkaMST::execute(adjList, vertexCount);
```

**Сложность:** O(E log V) — не более log V итераций

---

### Структура Union-Find (DSU)

Вспомогательная структура для эффективного отслеживания компонент связности:

```cpp
UnionFind uf(n);           // n элементов
uf.find(v);                // Найти корень компоненты
uf.merge(u, v);            // Объединить компоненты
uf.connected(u, v);        // Проверить связность
uf.componentCount();       // Количество компонент
```

**Оптимизации:**
- Сжатие путей (path compression)
- Объединение по рангу (union by rank)

**Сложность:** O(α(n)) ≈ O(1) амортизированная для всех операций

## Формат входных данных

### Список смежности (SpaceArray)

Граф хранится в формате пар (сосед, вес):
```cpp
SpaceArray<int> adjList;
// Добавление ребра (u, v) с весом w:
adjList.add(u, v);      // сосед
adjList.add(u, w);      // вес
adjList.add(v, u);      // обратное ребро для неориентированного графа
adjList.add(v, w);
```

### Список рёбер (FactorArray)

```cpp
FactorArray<WeightedEdge> edges;
edges.add(WeightedEdge(from, to, weight), edges.size());
```

## Формат выходных данных

```cpp
struct MSTResult {
    FactorArray<Edge> edges;  // Рёбра MST
    int totalWeight;          // Суммарный вес
    bool isConnected;         // Граф связный?
};

struct Edge {
    int v1;  // Первая вершина
    int v2;  // Вторая вершина
};
```

## Пример из лекции

Граф с 7 вершинами (A=0, B=1, C=2, D=3, E=4, F=5, G=6):

```
    A(0)
   / |  
  7  5   
 /   |    
B(1)-D(3)--F(5)
|\  9|   6/ |
| \  |   /  |
8 7 15  8  11
|   \ | /   |
C(2)-E(4)--G(6)
  5     9
```

MST: {A-D, C-E, D-F, A-B, B-E, E-G}, вес = 39

Все три алгоритма дают одинаковый результат (суммарный вес MST всегда одинаков для данного графа).

## 1. Введение и определения

### 1.1 Что такое остовное дерево?

**Остовное дерево** (spanning tree) графа G = (V, E) — это подграф T = (V, E'), который:
- Содержит **все вершины** исходного графа V
- Является **деревом** (связный граф без циклов)
- Содержит ровно **|V| - 1 рёбер**

```
Исходный граф:                 Одно из остовных деревьев:
    A ---3--- B                    A ---3--- B
    |\ 5    / |                    |         |
   1|  \  /4  |2                  1|         |2
    |   \/    |                    |         |
    D ---6--- C                    D         C

Рёбра графа: {A-B, A-C, A-D, B-C, D-C}     Рёбра дерева: {A-B, A-D, B-C}
Вес: 3+5+1+4+2+6 = 21                      Вес: 3+1+2 = 6
```

### 1.2 Минимальное остовное дерево

**Минимальное остовное дерево** (Minimum Spanning Tree, MST) — это остовное дерево с **минимальной суммой весов рёбер**.

Для взвешенного связного неориентированного графа MST всегда существует. Если все веса рёбер различны, то MST единственно.

### 1.3 Формальная постановка задачи

**Дано:**
- Неориентированный связный граф G = (V, E)
- Весовая функция w: E → ℝ

**Найти:**
- Подмножество рёбер T ⊆ E такое, что:
  1. (V, T) — связный граф (дерево)
  2. Σ w(e) для e ∈ T минимальна

### 1.4 Примеры из реальной жизни

| Задача | Вершины | Рёбра | Вес |
|--------|---------|-------|-----|
| Прокладка кабеля | Города | Возможные соединения | Стоимость прокладки |
| Электросеть | Подстанции | Линии электропередач | Длина провода |
| Дорожная сеть | Населённые пункты | Дороги | Стоимость строительства |
| Компьютерная сеть | Компьютеры | Кабели | Длина кабеля |

## 2. Свойства MST

### 2.1 Лемма о безопасном ребре (Cut Property)

**Разрез** (cut) графа — это разбиение множества вершин V на два непустых подмножества S и V \ S.

**Теорема (Cut Property):**
Пусть (S, V \ S) — произвольный разрез графа G. Если ребро e = (u, v) имеет **минимальный вес** среди всех рёбер, пересекающих разрез (u ∈ S, v ∈ V \ S), то e принадлежит некоторому MST.

```
Разрез {A, B} и {C, D}:

    A ----5---- B
    |           |
   2|           |3    ← ребро B-C (вес 3) — минимальное,
    |           |       пересекающее разрез
    D ----4---- C

Ребро B-C обязательно входит в MST.
```

### 2.2 Лемма о цикле (Cycle Property)

**Теорема (Cycle Property):**
Если в цикле графа G ребро e имеет **максимальный вес** (строго больше остальных), то e **не принадлежит** ни одному MST.

```
Цикл A-B-C-A:
    A
   / \
  1   3    ← ребро A-C (вес 3) — максимальное в цикле
 /     \
B --2-- C

MST не содержит ребро A-C.
```

### 2.3 Количество рёбер в MST

MST графа с V вершинами всегда содержит ровно **V - 1** ребро.

**Доказательство:** Дерево с n вершинами имеет n - 1 ребро (индукция по n).

### 2.4 Единственность MST

- Если **все веса различны** → MST единственно
- Если **есть равные веса** → может быть несколько MST с одинаковым суммарным весом

## 3. Структура данных Union-Find

### 3.1 Назначение

**Union-Find** (Disjoint Set Union, DSU) — структура данных для:
- Хранения множества **непересекающихся множеств**
- Быстрого **объединения** двух множеств
- Быстрой **проверки принадлежности** элементов одному множеству

В контексте MST: отслеживание компонент связности при добавлении рёбер.

### 3.2 Операции

| Операция | Описание | Применение в MST |
|----------|----------|------------------|
| `makeSet(x)` | Создать множество из одного элемента x | Инициализация: каждая вершина — отдельная компонента |
| `find(x)` | Найти представителя множества, содержащего x | Определить, в какой компоненте находится вершина |
| `union(x, y)` | Объединить множества, содержащие x и y | Добавить ребро (x, y) в MST |

### 3.3 Наивная реализация

```cpp
class UnionFindNaive {
    int parent[MAX_N];
    
public:
    void makeSet(int v) {
        parent[v] = v;  // Каждый элемент — свой родитель
    }
    
    int find(int v) {
        while (parent[v] != v) {
            v = parent[v];
        }
        return v;
    }
    
    void unite(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);
        if (rootA != rootB) {
            parent[rootA] = rootB;
        }
    }
};
```

**Проблема:** В худшем случае дерево вырождается в цепочку → find() работает за O(n).

```
После операций union(0,1), union(1,2), union(2,3), union(3,4):

    4
    |
    3
    |
    2
    |
    1
    |
    0

find(0) требует 4 шага!
```

### 3.4 Оптимизация 1: Сжатие путей (Path Compression)

**Идея:** При вызове find(x) все вершины на пути к корню напрямую присоединяются к корню.

```cpp
int find(int v) {
    if (parent[v] != v) {
        parent[v] = find(parent[v]);  // Рекурсивное сжатие
    }
    return parent[v];
}
```

**Эффект:**

```
До find(0):          После find(0):
    4                    4
    |                  / | \
    3                 3  2  1
    |                        |
    2                        0
    |
    1
    |
    0
```

### 3.5 Оптимизация 2: Объединение по рангу (Union by Rank)

**Идея:** При объединении меньшее дерево присоединяем к большему.

**Ранг** — оценка высоты дерева (изначально 0 для всех).

```cpp
void unite(int a, int b) {
    int rootA = find(a);
    int rootB = find(b);
    
    if (rootA == rootB) return;
    
    // Присоединяем меньшее к большему
    if (rank[rootA] < rank[rootB]) {
        parent[rootA] = rootB;
    } else if (rank[rootA] > rank[rootB]) {
        parent[rootB] = rootA;
    } else {
        parent[rootB] = rootA;
        rank[rootA]++;
    }
}
```

### 3.6 Итоговая сложность

С обеими оптимизациями:
- `find(x)`: **O(α(n))** — обратная функция Аккермана
- `union(x, y)`: **O(α(n))**

Функция α(n) растёт **невероятно медленно**: для всех практических значений n (до 10^80) α(n) ≤ 4.

**Практически**: O(1) на операцию.

### 3.7 Полная реализация

```cpp
class UnionFind {
    FactorArray<int> parent;
    FactorArray<int> rank;
    int components;

public:
    explicit UnionFind(int n) : components(n) {
        for (int i = 0; i < n; ++i) {
            parent.add(i, i);      // parent[i] = i
            rank.add(0, i);        // rank[i] = 0
        }
    }
    
    int find(int v) {
        if (parent.get(v) != v) {
            int root = find(parent.get(v));
            // Сжатие пути
            parent.remove(v);
            parent.add(root, v);
        }
        return parent.get(v);
    }
    
    bool merge(int u, int v) {
        int rootU = find(u);
        int rootV = find(v);
        
        if (rootU == rootV) return false;  // Уже в одном множестве
        
        // Объединение по рангу
        if (rank.get(rootU) < rank.get(rootV)) {
            std::swap(rootU, rootV);
        }
        
        parent.remove(rootV);
        parent.add(rootU, rootV);
        
        if (rank.get(rootU) == rank.get(rootV)) {
            int r = rank.get(rootU);
            rank.remove(rootU);
            rank.add(r + 1, rootU);
        }
        
        components--;
        return true;
    }
    
    bool connected(int u, int v) {
        return find(u) == find(v);
    }
    
    int componentCount() const {
        return components;
    }
};
```

## 4. Алгоритм Краскала

### 4.1 История

Алгоритм предложен Джозефом Краскалом в 1956 году. Является классическим примером **жадного алгоритма**.

### 4.2 Идея алгоритма

1. Отсортировать все рёбра по весу (по возрастанию)
2. Последовательно рассматривать рёбра от лёгких к тяжёлым
3. Добавить ребро в MST, если оно **не образует цикл**
4. Остановиться, когда добавлено V - 1 ребро

### 4.3 Псевдокод

```
Kruskal(G):
    MST = ∅
    отсортировать рёбра E по весу
    создать UnionFind для V вершин
    
    для каждого ребра (u, v, w) в порядке возрастания веса:
        если find(u) ≠ find(v):     // u и v в разных компонентах
            MST = MST ∪ {(u, v)}
            union(u, v)
            если |MST| = |V| - 1:
                break
    
    вернуть MST
```

### 4.4 Пример работы

```
Граф:
    A ---7--- B
    |        /|\
   5|      9/ | \7
    |      /  |  \
    D----15---E---5---C
     \       / \     /
      6    8/   \9  /8
       \  /      \ /
        F ---11--- G

Рёбра отсортированные: 
(C,E,5), (A,D,5), (D,F,6), (A,B,7), (B,E,7), (B,C,8), (E,F,8), 
(B,D,9), (E,G,9), (F,G,11), (D,E,15)

Шаг 1: (C,E,5) — добавляем, MST = {C-E}
Шаг 2: (A,D,5) — добавляем, MST = {C-E, A-D}
Шаг 3: (D,F,6) — добавляем, MST = {C-E, A-D, D-F}
Шаг 4: (A,B,7) — добавляем, MST = {C-E, A-D, D-F, A-B}
Шаг 5: (B,E,7) — добавляем, MST = {C-E, A-D, D-F, A-B, B-E}
       Теперь компоненты {A,B,C,D,E,F} объединены
Шаг 6: (B,C,8) — пропускаем (B и C уже связаны)
Шаг 7: (E,F,8) — пропускаем (E и F уже связаны)
Шаг 8: (B,D,9) — пропускаем (B и D уже связаны)
Шаг 9: (E,G,9) — добавляем, MST = {C-E, A-D, D-F, A-B, B-E, E-G}
       Добавлено 6 = V-1 рёбер, СТОП!

Итог: MST = {C-E, A-D, D-F, A-B, B-E, E-G}
Вес = 5 + 5 + 6 + 7 + 7 + 9 = 39
```

### 4.5 Визуализация процесса

```
Начало:        После (C,E):   После (A,D):   После (D,F):
A   B          A   B          A   B          A   B
                                |              |
D   E   C      D   E---C       D   E---C       D   E---C
                                               |
F   G          F   G           F   G           F   G


После (A,B):   После (B,E):   После (E,G):   [ГОТОВО]
A---B          A---B          A---B
|              |   |          |   |
D   E---C      D   E---C      D   E---C
|              |   |          |   |

               [объединены]       G
```

### 4.6 Реализация

```cpp
class KruskalMST {
public:
    static MSTResult execute(const FactorArray<WeightedEdge>& edges, 
                             int vertexCount) {
        MSTResult result;
        
        if (vertexCount <= 1) {
            result.isConnected = true;
            return result;
        }
        
        // Копируем и сортируем рёбра
        FactorArray<WeightedEdge> sortedEdges;
        for (int i = 0; i < edges.size(); ++i) {
            sortedEdges.add(edges.get(i), sortedEdges.size());
        }
        sortEdges(sortedEdges);
        
        // Union-Find
        UnionFind uf(vertexCount);
        
        int edgesAdded = 0;
        int targetEdges = vertexCount - 1;
        
        for (int i = 0; i < sortedEdges.size() && edgesAdded < targetEdges; ++i) {
            WeightedEdge edge = sortedEdges.get(i);
            
            // Если вершины в разных компонентах — добавляем ребро
            if (uf.merge(edge.from, edge.to)) {
                result.edges.add(edge.toEdge(), result.edges.size());
                result.totalWeight += edge.weight;
                edgesAdded++;
            }
        }
        
        result.isConnected = (edgesAdded == targetEdges);
        return result;
    }
    
private:
    // Сортировка вставками O(E²)
    static void sortEdges(FactorArray<WeightedEdge>& edges) {
        for (int i = 1; i < edges.size(); ++i) {
            WeightedEdge key = edges.get(i);
            int j = i - 1;
            
            while (j >= 0 && edges.get(j).weight > key.weight) {
                WeightedEdge temp = edges.get(j);
                edges.remove(j + 1);
                edges.add(temp, j + 1);
                j--;
            }
            edges.remove(j + 1);
            edges.add(key, j + 1);
        }
    }
};
```

### 4.7 Анализ сложности

| Этап | Сложность | Примечание |
|------|-----------|------------|
| Сортировка рёбер | O(E log E) | Доминирующая операция |
| Цикл по рёбрам | O(E) | E итераций |
| Операции Union-Find | O(E · α(V)) ≈ O(E) | α(V) ≈ константа |
| **Итого** | **O(E log E)** | = O(E log V), т.к. E ≤ V² |

**Память:** O(V + E) — хранение рёбер и Union-Find.

### 4.8 Когда использовать Краскала?

**Преимущества:**
- Простая реализация
- Эффективен для **разреженных графов** (E ≈ V)
- Легко параллелизуется (сортировка)
- Работает с графом, заданным списком рёбер

**Недостатки:**
- Требует сортировки всех рёбер
- Менее эффективен для плотных графов (E ≈ V²)

## 5. Алгоритм Прима

### 5.1 История

Алгоритм разработан Войцехом Ярником в 1930 году, переоткрыт Робертом Примом в 1957 году и Эдсгером Дейкстрой в 1959 году.

### 5.2 Идея алгоритма

1. Начать с произвольной вершины (обычно 0)
2. «Растить» дерево, добавляя на каждом шаге **минимальное ребро**, соединяющее дерево с внешней вершиной
3. Повторять, пока все вершины не будут включены

**Отличие от Краскала:** Прим растит **одно дерево** из начальной точки, Краскал может одновременно растить **лес** из нескольких деревьев.

### 5.3 Псевдокод

```
Prim(G, start):
    key[v] = ∞ для всех v          // Минимальный вес ребра к дереву
    key[start] = 0
    parent[v] = NIL для всех v
    inMST[v] = false для всех v
    
    повторить V раз:
        u = вершина с минимальным key[u], где inMST[u] = false
        inMST[u] = true
        
        для каждого соседа v вершины u:
            если inMST[v] = false и w(u,v) < key[v]:
                key[v] = w(u,v)
                parent[v] = u
    
    MST = {(v, parent[v]) для всех v ≠ start}
    вернуть MST
```

### 5.4 Пример работы

```
Граф:
    A ---7--- B
    |        /|
   5|      9/ |7
    |      /  |
    D----15---E---5---C

Начинаем с A:

Шаг 0: key = [0, ∞, ∞, ∞, ∞], inMST = [F, F, F, F, F]
       Выбираем A (key=0), добавляем в MST
       Обновляем соседей: key[B]=7, key[D]=5
       
Шаг 1: key = [0, 7, ∞, 5, ∞], inMST = [T, F, F, F, F]
       Выбираем D (key=5), добавляем ребро A-D
       Обновляем: key[E]=15 (через D-E)
       
Шаг 2: key = [0, 7, ∞, 5, 15], inMST = [T, F, F, T, F]
       Выбираем B (key=7), добавляем ребро A-B
       Обновляем: key[E]=min(15,7)=7, key[C]=∞
       
Шаг 3: key = [0, 7, ∞, 5, 7], inMST = [T, T, F, T, F]
       Выбираем E (key=7), добавляем ребро B-E
       Обновляем: key[C]=5
       
Шаг 4: key = [0, 7, 5, 5, 7], inMST = [T, T, F, T, T]
       Выбираем C (key=5), добавляем ребро E-C
       
Итог: MST = {A-D, A-B, B-E, E-C}
```

### 5.5 Визуализация «роста» дерева

```
Шаг 0:         Шаг 1:         Шаг 2:         Шаг 3:         Шаг 4:
  [A]            A               A---B          A---B          A---B
   ↓             |               |              |   |          |   |
(start)          D               D              D   E          D   E---C
                                                               
Вес: 0         +5 = 5         +7 = 12        +7 = 19        +5 = 24
```

### 5.6 Реализация (с массивом)

```cpp
class PrimMST {
public:
    static MSTResult execute(const SpaceArray<int>& adjList, int vertexCount) {
        MSTResult result;
        
        if (vertexCount <= 1) {
            result.isConnected = (vertexCount >= 0);
            return result;
        }
        
        const int INF = std::numeric_limits<int>::max();
        
        // key[v] = минимальный вес ребра, соединяющего v с деревом
        FactorArray<int> key;
        // parent[v] = родитель v в MST
        FactorArray<int> parent;
        // inMST[v] = true, если v уже в дереве
        FactorArray<bool> inMST;
        
        // Инициализация
        for (int i = 0; i < vertexCount; ++i) {
            key.add(INF, i);
            parent.add(-1, i);
            inMST.add(false, i);
        }
        
        // Начинаем с вершины 0
        key.remove(0);
        key.add(0, 0);
        
        for (int count = 0; count < vertexCount; ++count) {
            // Находим вершину с минимальным key среди не включённых
            int u = -1;
            int minKey = INF;
            
            for (int v = 0; v < vertexCount; ++v) {
                if (!inMST.get(v) && key.get(v) < minKey) {
                    minKey = key.get(v);
                    u = v;
                }
            }
            
            if (u == -1) break;  // Граф несвязный
            
            // Включаем u в MST
            inMST.remove(u);
            inMST.add(true, u);
            
            // Добавляем ребро в результат (кроме стартовой вершины)
            if (parent.get(u) != -1) {
                Edge edge;
                edge.v1 = parent.get(u);
                edge.v2 = u;
                result.edges.add(edge, result.edges.size());
                result.totalWeight += key.get(u);
            }
            
            // Обновляем ключи соседей
            if (u < adjList.blockCount()) {
                int blockSize = adjList.blockSize(u);
                for (int i = 0; i < blockSize; i += 2) {
                    int v = adjList.get(u, i);
                    int weight = adjList.get(u, i + 1);
                    
                    if (!inMST.get(v) && weight < key.get(v)) {
                        key.remove(v);
                        key.add(weight, v);
                        parent.remove(v);
                        parent.add(u, v);
                    }
                }
            }
        }
        
        result.isConnected = (result.edges.size() == vertexCount - 1);
        return result;
    }
};
```

### 5.7 Анализ сложности

**Реализация с массивом (как выше):**

| Этап | Сложность |
|------|-----------|
| Инициализация | O(V) |
| Внешний цикл | V итераций |
| Поиск минимума | O(V) на каждой итерации |
| Обновление соседей | O(степень вершины) |
| **Итого** | **O(V²)** |

**Реализация с бинарной кучей:**

| Этап | Сложность |
|------|-----------|
| Извлечение минимума | O(log V) |
| Обновление ключа | O(log V) |
| Всего операций | O(E) |
| **Итого** | **O(E log V)** |

**Реализация с кучей Фибоначчи:**

| Операция | Сложность |
|----------|-----------|
| Извлечение минимума | O(log V) амортизированно |
| Уменьшение ключа | O(1) амортизированно |
| **Итого** | **O(E + V log V)** |

### 5.8 Когда использовать Прима?

**Преимущества:**
- Эффективен для **плотных графов** (E ≈ V²)
- Версия O(V²) проста в реализации без дополнительных структур
- Работает с графом в виде матрицы смежности

**Недостатки:**
- Требует хранить граф с быстрым доступом к соседям
- Оптимальные версии сложнее в реализации

## 6. Алгоритм Борувки

### 6.1 История

Алгоритм предложен Отакаром Борувкой в **1926 году** — это **первый** алгоритм для построения MST! Разработан для проектирования эффективной электрической сети в Моравии (Чехословакия).

### 6.2 Идея алгоритма

1. Изначально каждая вершина — отдельная компонента (лес из V деревьев)
2. На каждой **итерации (фазе)**:
   - Для **каждой компоненты** найти минимальное ребро, ведущее наружу
   - Добавить все найденные рёбра в MST
   - Объединить соответствующие компоненты
3. Повторять, пока не останется одна компонента

### 6.3 Ключевое наблюдение

На каждой итерации количество компонент уменьшается **минимум вдвое**:
- Каждая компонента добавляет хотя бы одно ребро наружу
- Ребро соединяет две компоненты → они объединяются

**Следствие:** Максимум **O(log V)** итераций.

### 6.4 Псевдокод

```
Boruvka(G):
    создать UnionFind для V вершин
    MST = ∅
    
    пока componentCount > 1:
        // Для каждой компоненты найти минимальное внешнее ребро
        cheapest[c] = NIL для всех компонент c
        
        для каждого ребра (u, v, w):
            compU = find(u)
            compV = find(v)
            
            если compU ≠ compV:
                если cheapest[compU] = NIL или w < вес(cheapest[compU]):
                    cheapest[compU] = (u, v, w)
                если cheapest[compV] = NIL или w < вес(cheapest[compV]):
                    cheapest[compV] = (u, v, w)
        
        // Добавить все минимальные рёбра
        для каждой компоненты c:
            если cheapest[c] ≠ NIL:
                (u, v, w) = cheapest[c]
                если find(u) ≠ find(v):  // Проверка на дубликаты
                    MST = MST ∪ {(u, v)}
                    union(u, v)
    
    вернуть MST
```

### 6.5 Пример работы

```
Граф (8 вершин):
  0 --1-- 1 --6-- 2 --3-- 3
  |       |       |       |
  7       2       5       4
  |       |       |       |
  4 --8-- 5 --9-- 6 --10- 7

Итерация 1:
  Компонента {0}: мин. ребро 0-1 (вес 1)
  Компонента {1}: мин. ребро 0-1 (вес 1)
  Компонента {2}: мин. ребро 2-3 (вес 3)
  Компонента {3}: мин. ребро 2-3 (вес 3)
  Компонента {4}: мин. ребро 0-4 (вес 7)
  Компонента {5}: мин. ребро 1-5 (вес 2)
  Компонента {6}: мин. ребро 2-6 (вес 5)
  Компонента {7}: мин. ребро 3-7 (вес 4)
  
  Добавляем: 0-1, 2-3, 0-4, 1-5, 2-6, 3-7
  После слияния: {0,1,4,5}, {2,3,6,7}

  Добавляем: 1-2
  После слияния: {0,1,2,3,4,5,6,7} — одна компонента, СТОП!
```


### 6.7 Реализация

```cpp
class BoruvkaMST {
public:
    static MSTResult execute(const FactorArray<WeightedEdge>& edges, 
                             int vertexCount) {
        MSTResult result;
        
        if (vertexCount <= 1) {
            result.isConnected = (vertexCount >= 0);
            return result;
        }
        
        const int INF = std::numeric_limits<int>::max();
        UnionFind uf(vertexCount);
        
        int numComponents = vertexCount;
        int targetEdges = vertexCount - 1;
        
        // Пока больше одной компоненты
        while (numComponents > 1 && result.edges.size() < targetEdges) {
            
            // cheapest[i] = индекс минимального внешнего ребра для компоненты i
            FactorArray<int> cheapest;
            for (int i = 0; i < vertexCount; ++i) {
                cheapest.add(-1, i);
            }
            
            // Найти минимальное ребро для каждой компоненты
            for (int i = 0; i < edges.size(); ++i) {
                WeightedEdge edge = edges.get(i);
                int compU = uf.find(edge.from);
                int compV = uf.find(edge.to);
                
                if (compU == compV) continue;  // Внутреннее ребро
                
                // Обновляем cheapest для обеих компонент
                if (cheapest.get(compU) == -1 || 
                    edge.weight < edges.get(cheapest.get(compU)).weight) {
                    cheapest.remove(compU);
                    cheapest.add(i, compU);
                }
                
                if (cheapest.get(compV) == -1 || 
                    edge.weight < edges.get(cheapest.get(compV)).weight) {
                    cheapest.remove(compV);
                    cheapest.add(i, compV);
                }
            }
            
            // Добавляем найденные рёбра
            bool addedAny = false;
            for (int v = 0; v < vertexCount; ++v) {
                int edgeIdx = cheapest.get(v);
                if (edgeIdx == -1) continue;
                
                WeightedEdge edge = edges.get(edgeIdx);
                
                // Проверяем, не добавлено ли уже
                if (uf.merge(edge.from, edge.to)) {
                    result.edges.add(edge.toEdge(), result.edges.size());
                    result.totalWeight += edge.weight;
                    numComponents--;
                    addedAny = true;
                }
            }
            
            if (!addedAny) break;  // Граф несвязный
        }
        
        result.isConnected = (result.edges.size() == targetEdges);
        return result;
    }
};
```

### 6.8 Анализ сложности

| Этап | Сложность |
|------|-----------|
| Количество итераций | O(log V) |
| Одна итерация: проход по рёбрам | O(E) |
| Операции Union-Find | O(E · α(V)) |
| **Итого** | **O(E log V)** |

**Память:** O(V + E)

### 6.9 Когда использовать Борувку?

**Преимущества:**
- Хорошо **параллелизуется** (каждая компонента независима)
- Эффективен для параллельных/распределённых вычислений
- Исторически первый алгоритм MST

**Недостатки:**
- Сложнее в последовательной реализации, чем Краскал
- На практике часто проигрывает оптимизированным версиям Прима

## 7. Сравнение алгоритмов

### 7.1 Сводная таблица сложности

| Алгоритм | Время | Память | Структура данных |
|----------|-------|--------|------------------|
| Краскал | O(E log E) | O(V + E) | Union-Find |
| Прим (массив) | O(V²) | O(V) | Массив |
| Прим (бин. куча) | O(E log V) | O(V) | Бинарная куча |
| Прим (Фибоначчи) | O(E + V log V) | O(V) | Куча Фибоначчи |
| Борувка | O(E log V) | O(V + E) | Union-Find |

### 7.2 Рекомендации по выбору

```
                    Плотность графа
                    |
        Разреженный |  Средний  |   Плотный
        E ≈ V       |  E ≈ V√V  |   E ≈ V²
        ------------+-----------+------------
                    |           |
Обычный:  Краскал   |  Краскал  |  Прим O(V²)
                    |           |
Параллельный:       Борувка для всех случаев
```

### 7.3 Практические соображения

1. **Краскал** — универсальный выбор для большинства задач
2. **Прим O(V²)** — когда граф плотный И нужна простая реализация
3. **Борувка** — когда доступны параллельные ресурсы
4. **Прим с кучей** — для разреженных графов при наличии готовой реализации кучи


## 8. Практические применения

### 8.1 Проектирование сетей

**Задача:** Соединить N городов оптоволоконной сетью с минимальной стоимостью.

```cpp
struct City {
    string name;
    double latitude, longitude;
};

// Вычислить расстояние между городами
double distance(const City& a, const City& b);

// Построить полный граф и найти MST
FactorArray<WeightedEdge> buildNetwork(const vector<City>& cities) {
    FactorArray<WeightedEdge> edges;
    
    // Добавить все возможные соединения
    for (int i = 0; i < cities.size(); ++i) {
        for (int j = i + 1; j < cities.size(); ++j) {
            int cost = static_cast<int>(distance(cities[i], cities[j]) * 1000);
            edges.add(WeightedEdge(i, j, cost), edges.size());
        }
    }
    
    MSTResult mst = KruskalMST::execute(edges, cities.size());
    return mst.edges;
}
```

### 8.2 Кластеризация данных

**Идея:** Построить MST на точках данных, затем удалить k-1 самое тяжёлое ребро → получить k кластеров.

```cpp
vector<int> clusterData(const vector<Point>& points, int k) {
    // Строим MST
    MSTResult mst = buildMST(points);
    
    // Сортируем рёбра MST по убыванию веса
    sortByWeightDescending(mst.edges);
    
    // Удаляем k-1 самых тяжёлых рёбер
    UnionFind uf(points.size());
    for (int i = k - 1; i < mst.edges.size(); ++i) {
        Edge e = mst.edges.get(i);
        uf.merge(e.v1, e.v2);
    }
    
    // Возвращаем метки кластеров
    vector<int> labels(points.size());
    for (int i = 0; i < points.size(); ++i) {
        labels[i] = uf.find(i);
    }
    return labels;
}
```

### 8.3 Приближённое решение задачи коммивояжёра

**TSP-MST эвристика:**
1. Построить MST
2. Обойти MST в порядке DFS
3. Пропустить уже посещённые вершины

Даёт решение не более чем в 2 раза хуже оптимального (для метрического TSP).

### 8.4 Сегментация изображений

**Алгоритм Фельзенсвальба-Хаттенлохера:**
1. Каждый пиксель — вершина
2. Ребро между соседними пикселями с весом = разница яркости
3. Модифицированный MST для объединения похожих регионов

## Заключение

Минимальное остовное дерево — фундаментальная задача теории графов с множеством практических применений. Три классических алгоритма (Краскал, Прим, Борувка) демонстрируют разные подходы:

- **Краскал** — жадный выбор по глобально отсортированным рёбрам
- **Прим** — «рост» дерева из одной точки
- **Борувка** — параллельное объединение компонент

Все три алгоритма основаны на **свойстве разреза** (Cut Property) и гарантируют оптимальное решение.

**Ключевой навык:** Понимание, когда какой алгоритм эффективнее, и умение адаптировать базовые версии под конкретные задачи.

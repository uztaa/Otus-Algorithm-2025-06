## Введение {#введение}

**Динамическое программирование (ДП)** — это метод решения задач, основанный на разбиении сложной задачи на перекрывающиеся подзадачи и сохранении результатов (мемоизация) для избежания повторных вычислений.

### История
- **Термин** введён Ричардом Беллманом в 1950-х годах
- **Назначение**: оптимизация больших комбинаторных проблем
- **Применение**: теория управления, экономика, информатика

### Когда использовать ДП?
1. **Оптимальная подструктура** — оптимальное решение состоит из оптимальных решений подзадач
2. **Перекрывающиеся подзадачи** — одни и те же подзадачи решаются многократно
3. **Независимость подзадач** — решение одной не зависит от решения другой

## Основные принципы {#принципы}

### Принцип 1: Рекуррентное соотношение
ДП основано на том, что решение задачи выражается через решения подзадач:

```
dp[n] = f(dp[n-1], dp[n-2], ..., dp[0])
```

**Пример**: числа Фибоначчи
```
f(n) = f(n-1) + f(n-2)
f(0) = 0, f(1) = 1
```

### Принцип 2: Базовые случаи
Каждое рекуррентное соотношение требует базовых случаев (граничные условия):

```
dp[0] = начальное_значение
dp[1] = начальное_значение
```

### Принцип 3: Направление вычисления

#### **Top-Down (Мемоизация)**
- Начинаем с больших подзадач
- Разбиваем на меньшие
- Кэшируем результаты
- Рекурсивный подход

```cpp
map<int, long long> memo;

long long fib(int n) {
    if (n <= 1) return n;
    if (memo.count(n)) return memo[n];
    return memo[n] = fib(n-1) + fib(n-2);
}
```

#### **Bottom-Up (Табуляция)**
- Начинаем с базовых случаев
- Строим решение от простого к сложному
- Используем таблицу/массив
- Итеративный подход

```cpp
vector<long long> dp(n+1);
dp[0] = 0;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
    dp[i] = dp[i-1] + dp[i-2];
}
```

### Сравнение подходов

| Аспект | Top-Down | Bottom-Up |
|--------|----------|-----------|
| Подход | Рекурсия + кэш | Итерация |
| Интуитивность | Выше | Ниже |
| Производительность | O(2^n) → O(n) | O(n) |
| Память стека | Глубина O(n) | O(1) |
| Реализация | Проще | Проще (после понимания) |

## Классические задачи {#классические}

### Задача 1: Числа Фибоначчи

**Определение**: F(n) = F(n-1) + F(n-2), F(0)=0, F(1)=1

**Наивный подход**: O(2^n) — экспоненциально медленно
```
F(5) вызывает F(4) и F(3)
F(4) вызывает F(3) и F(2)
F(3) вызывает F(2) и F(1)
...
F(3) вычисляется многократно!
```

**ДП решение**: O(n)
```cpp
dp[0] = 0;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
    dp[i] = dp[i-1] + dp[i-2];
}
```

### Задача 2: Монеты (Coin Change)

**Проблема**: Найти минимальное количество монет номиналов c₁, c₂, ..., cₖ для составления суммы S.

**Рекуррентное соотношение**:
```
dp[i] = минимум количество монет для суммы i

dp[i] = 1 + min(dp[i - c_j]) для всех доступных монет c_j
```

**Базовый случай**:
```
dp[0] = 0  (сумма 0 требует 0 монет)
dp[i] = INF для остальных (изначально невозможно)
```

**Пример**:
```
Монеты: [1, 2, 5], Сумма: 5

dp[0] = 0
dp[1] = 1 (монета 1)
dp[2] = 1 (монета 2)
dp[3] = 2 (2+1)
dp[4] = 2 (2+2)
dp[5] = 1 (монета 5)
```

**Код**:
```cpp
vector<int> dp(sum + 1, INT_MAX);
dp[0] = 0;

for (int i = 1; i <= sum; ++i) {
    for (int coin : coins) {
        if (coin <= i && dp[i - coin] != INT_MAX) {
            dp[i] = min(dp[i], 1 + dp[i - coin]);
        }
    }
}
```

### Задача 3: Рюкзак (Knapsack)

**Проблема 0/1 Knapsack**: Найти максимальную стоимость предметов, вмещающихся в рюкзак ёмкостью W.
- Каждый предмет можно взять максимум один раз
- Вес каждого предмета w[i], стоимость v[i]

**Рекуррентное соотношение**:
```
dp[w] = максимальная стоимость при ёмкости w

dp[w] = max(
    dp[w],                          // не берём i-й предмет
    dp[w - weight[i]] + value[i]    // берём i-й предмет
)
```

**2D таблица**:
```
dp[i][w] = максимальная стоимость используя первые i предметов с ёмкостью w

dp[i][w] = max(
    dp[i-1][w],                          // без i-го предмета
    dp[i-1][w-weight[i]] + value[i]     // с i-м предметом (если помещается)
)
```

### Задача 4: Наибольшая общая подпоследовательность (LCS)

**Проблема**: Найти максимальную длину подпоследовательности, общей для двух строк.

**Подпоследовательность** (не обязательно подряд):
- Строка "ABCDGH" → подпоследовательность "ACH", "AG", "ADH" и т.д.

**Рекуррентное соотношение**:
```
dp[i][j] = длина LCS для первых i символов строки1 и j символов строки2

Если str1[i-1] == str2[j-1]:
    dp[i][j] = 1 + dp[i-1][j-1]  // добавляем совпадающий символ

Иначе:
    dp[i][j] = max(
        dp[i-1][j],   // пропускаем символ из str1
        dp[i][j-1]    // пропускаем символ из str2
    )
```

**Базовые случаи**:
```
dp[0][j] = 0 (пустая первая строка)
dp[i][0] = 0 (пустая вторая строка)
```

**Пример**:
```
str1 = "AGGTAB"
str2 = "GXTXAYB"

Таблица:
    ""  G  X  T  X  A  Y  B
""   0  0  0  0  0  0  0  0
A    0  0  0  0  0  1  1  1
G    0  1  1  1  1  1  1  1
G    0  1  1  1  1  1  1  1
T    0  1  1  2  2  2  2  2
A    0  1  1  2  2  3  3  3
B    0  1  1  2  2  3  3  4

LCS = 4 ("GTAB")
```

## Техники оптимизации {#техники}

### Техника 1: Мемоизация (Top-Down)

**Суть**: Кэширование результатов рекурсивных вызовов

```cpp
unordered_map<long long, long long> memo;

long long solve(int n, int k) {
    if (n == 0) return 1;
    if (n < 0) return 0;
    
    long long key = (long long)n * 100000 + k;
    if (memo.find(key) != memo.end()) {
        return memo[key];
    }
    
    long long result = solve(n-1, k) + solve(n-k, k);
    memo[key] = result;
    return result;
}
```

**Преимущества**:
- Вычисляются только нужные подзадачи
- Интуитивный рекурсивный подход

**Недостатки**:
- Глубина рекурсии может привести к переполнению стека
- Медленнее из-за вызовов функций

### Техника 2: Табуляция (Bottom-Up)

**Суть**: Итеративное заполнение таблицы от простого к сложному

```cpp
vector<vector<long long>> dp(n+1, vector<long long>(k+1, 0));

// Базовые случаи
for (int j = 0; j <= k; ++j) {
    dp[0][j] = 1;
}

// Заполнение таблицы
for (int i = 1; i <= n; ++i) {
    for (int j = 1; j <= k; ++j) {
        dp[i][j] = dp[i-1][j] + dp[i][j-1];
    }
}
```

**Преимущества**:
- Гарантированно O(n) временная сложность
- Нет глубокой рекурсии
- Часто быстрее на практике

**Недостатки**:
- Может вычислять ненужные подзадачи
- Требует больше памяти для таблицы

### Техника 3: Оптимизация памяти

Если DP зависит только от предыдущих строк/столбцов, можно уменьшить размер массива:

**Вместо O(n×m) памяти используем O(m)**:
```cpp
vector<long long> prev(m, 0), curr(m, 0);

for (int i = 1; i <= n; ++i) {
    for (int j = 1; j <= m; ++j) {
        curr[j] = prev[j] + curr[j-1];
    }
    swap(prev, curr);
}
```

### Техника 4: Восстановление решения

Часто нужно не только найти оптимальное значение, но и восстановить само решение:

```cpp
// Таблица для восстановления пути
vector<vector<int>> parent(n+1, vector<int>(m+1, -1));

// ...заполнение DP...

// Восстановление пути
vector<int> path;
int i = n, j = m;
while (i > 0 || j > 0) {
    path.push_back(parent[i][j]);
    // Переходим к следующей клетке согласно parent
    ...
}
reverse(path.begin(), path.end());
```

## Примеры решения {#примеры}

### Пример 1: Уход за домом (ДП по времени)

**Задача**: Хозяйка может убирать, стирать или гулять в каждый день. Умения улучшаются:
- Уборка: +1 навык уборки, -1 стирка
- Стирка: +1 навык стирки, -1 уборка
- Гулять: оба навыка падают на 20%

Максимизировать сумму навыков за N дней.

**Решение ДП**:
```cpp
dp[day][clean_skill][wash_skill] = 
    максимальная сумма после этого дня
```

### Пример 2: Макаронник (ДП по подмножествам)

**Задача**: Выбрать подмножество из n блюд (каждое один раз) максимального вкуса, но вес ≤ W.

**Решение 0/1 Knapsack**: см. выше

### Пример 3: Матрица полёта (ДП по маршруту)

**Задача**: Найти путь от верхнего левого угла матрицы к нижнему правому с максимальной суммой.

**Рекуррентное соотношение**:
```
dp[i][j] = максимальная сумма в клетке (i,j)

dp[i][j] = matrix[i][j] + max(dp[i-1][j], dp[i][j-1])
```

**Базовые случаи**:
```
dp[0][0] = matrix[0][0]
dp[0][j] = dp[0][j-1] + matrix[0][j]  (только справа)
dp[i][0] = dp[i-1][0] + matrix[i][0]  (только снизу)
```

## ДЗ-21: Задачи для решения {#дз-21}

### JUNIOR часть (4 задачи × 1 байт)

#### Задача 1: Раз горох, два горох (Дроби)
**Тип**: ДП или арифметика
**Суть**: Сложить две дроби a/b + c/d и вернуть несократимую дробь

**Ключевые моменты**:
- Привести к общему знаменателю
- Найти НОД для сокращения (Евклид)
- Результат несократимый

#### Задача 2: Ёлочка программиста (ДП на матрице)
**Тип**: ДП по маршруту (Triangle)
**Суть**: Найти гирлянду (путь) с максимальной суммой цифр

**Рекуррентное соотношение**:
```
dp[i][j] = максимальная сумма до позиции (i,j)
dp[i][j] = tree[i][j] + max(dp[i-1][j-1], dp[i-1][j])
```

**Направление**: Снизу вверх (от листьев к корню)

#### Задача 3: Пятью восемь (Подсчёт последовательностей)
**Тип**: ДП на последовательности
**Суть**: N-значные числа из цифр {5, 8}, без трёх подряд

**Рекуррентное соотношение**:
```
Состояние: (позиция, последние_две_цифры)

dp[i][last1][last2] = кол-во способов заполнить i позиций,
                       если последние две: last1, last2

dp[i][last1][last2] = 
    dp[i+1][5][last1] + dp[i+1][8][last1]
    (если не три подряд)
```

#### Задача 4: Большой остров (Поиск компонент)
**Тип**: ДП/BFS/DFS (ДП как классификация)
**Суть**: Найти количество связных компонент единиц в матрице

**Алгоритм**:
1. Проходим по каждой клетке
2. Если видим 1 и не посещали — DFS/BFS и счётчик++
3. Помечаем все посещённые как 0

### MIDDLE часть (Большой сарай - 4 этапа × 1-2 байта)

**Этап 1: Маленький сарай** (брутфорс, +1 байт)
- Размер N×M ≤ 30×30
- Перебор всех прямоугольников O(N²M²)
- Проверка каждого на свободность O(NM)

**Этап 2: Длина сарая** (+2 байта)
- Для каждой клетки найти высоту свободных клеток вверх
- ДП: `height[i][j] = height[i-1][j] + 1` если свободна

**Этап 3: Ширина сарая** (+1 байт)
- Для каждой позиции найти левую и правую границу
- Где высота ≥ текущей высоте
- Stack-based алгоритм или ДП

**Этап 4: Большой сарай** (+2 байта)
- Объединить все этапы
- Для каждой строки использовать Largest Rectangle in Histogram
- Сложность O(N×M)

## Практические советы

1. **Всегда проверяйте границы** — edge cases с 0, 1 элементом
2. **Используйте разумные начальные значения** — INF для минимума, -INF для максимума
3. **Оптимизируйте память** если возможно (rolliing array)
4. **Тестируйте на небольших примерах** прежде чем на больших
5. **Документируйте рекуррентное соотношение** перед кодированием

## Сложность анализ

| Задача | Подход | Наивно | С ДП | Улучшение |
|--------|--------|--------|------|-----------|
| Фибоначчи | Рекурсия | O(2^n) | O(n) | Экспоненциально |
| Knapsack | Перебор | O(2^n) | O(nW) | Существенно |
| LCS | Перебор | O(n×2^m) | O(n×m) | Значительно |
| Монеты | Перебор | O(2^n) | O(n×m) | Критично |

---

**Помните**: ДП — это не магия, а систематический способ избежать повторных вычислений путём сохранения промежуточных результатов.

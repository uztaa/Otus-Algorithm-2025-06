## Реализованные методы

### BinarySearchTree
- `void insert(int x)` — вставка элемента
- `bool search(int x)` — поиск элемента
- `void remove(int x)` — удаление элемента

### AVLTree (расширяет BST)
- `smallLeftRotation()` — малое левое вращение
- `smallRightRotation()` — малое правое вращение
- `bigLeftRotation()` — большое левое вращение
- `bigRightRotation()` — большое правое вращение
- `rebalance()` — перебалансировка

### Treap (расширяет BST)
- `merge(Tree l, Tree r)` — объединение деревьев
- `split(Tree t, int x)` — разделение дерева
- `insert(int x)` — вставка через split/merge
- `remove(int x)` — удаление через split/merge

## Результаты тестирования

### Условия
- N = 50000
- Вставка N чисел (случайный / возрастающий порядок)
- Поиск N/10 случайных чисел
- Удаление N/10 случайных элементов

### Таблица производительности

| Tree | Data | Insert | Search | Remove | Height | Size |
|------|------|--------|--------|--------|--------|------|
| BinarySearchTree | random | 17.0 ms | 1.1 ms | 1.8 ms | 37 | 47529 |
| BinarySearchTree | sorted | 22.3 s | 1.9 s | 3.7 s | 50000 | 50000 |
| AVLTree | random | 9.9 ms | 534.3 µs | 1.1 ms | 19 | 47529 |
| AVLTree | sorted | 5.9 ms | 137.7 µs | 530.7 µs | 16 | 50000 |
| Treap | random | 19.3 ms | 641.1 µs | 2.1 ms | 37 | 47529 |
| Treap | sorted | 6.5 ms | 134.2 µs | 279.0 µs | 36 | 50000 |

## Выводы

### 1. BST на отсортированных данных — катастрофа
- Высота дерева = N (50000) — полное вырождение в связный список
- Insert в **1300 раз** медленнее чем на случайных данных (22.3s vs 17ms)
- Все операции O(n) вместо O(log n)

### 2. AVL-дерево — гарантированная производительность
- Высота всегда O(log n): 19 для random, 16 для sorted
- На sorted данных даже **быстрее** чем на random (меньше перебалансировок после построения)
- Лучший выбор когда нужна предсказуемость

### 3. Treap — хороший компромисс
- Вероятностная балансировка работает: высота 36-37 при N=50000
- На sorted данных работает так же хорошо как AVL
- Проще в реализации чем AVL, поддерживает эффективные split/merge

### 4. Практические рекомендации

| Сценарий | Рекомендация |
|----------|--------------|
| Данные приходят случайно | BST или Treap |
| Данные могут быть упорядочены | AVL или Treap |
| Нужны split/merge операции | Treap |
| Критична worst-case производительность | AVL |

# ДЗ-11: Деревья Оптимального Поиска (ДОП)

## Описание

Реализация двух приближённых алгоритмов построения деревьев оптимального поиска:

- **Алгоритм 1**: Сортировка по весам → вставка в BST
- **Алгоритм 2**: Метод "центра тяжести" (рекурсивное разбиение)

## Алгоритмы

### Алгоритм 1 (плохой)

1. Отсортировать ключи по весам (убывание)
2. Вставлять в обычное BST в этом порядке

**Сложность:** O(n²) в худшем случае
**Результат:** Близок к случайному дереву

### Алгоритм 2 (хороший)

1. Отсортировать ключи по значению
2. Найти "центр тяжести" — ключ с минимальной разностью весов поддеревьев
3. Рекурсивно применить для поддеревьев

**Сложность:** O(n log n)
**Результат:** Асимптотически приближается к оптимальному

## Архитектурное решение

### Осознанный компромисс (нарушение LSP)

`OptimalTreeBase` наследует `IBinaryTree`, но методы `insert()` и `remove()` 
выбрасывают исключения, так как ДОП — **статическое** дерево.

**Почему так:**
- Совместимость с существующим кодом ДЗ-10 (единый интерфейс для бенчмарков)
- Минимальный рефакторинг
- Учебный проект — допустимый компромисс

**В production-коде** следует использовать отдельную иерархию:
```cpp
ISearchableTree (базовый)
├── IDynamicTree → BST, AVL, Treap
└── IStaticTree  → OptimalTree
```

## Бенчмарк

Сравниваются 5 деревьев на данных с Zipf-распределением весов:

| Дерево | Тип | Описание |
|--------|-----|----------|
| BinarySearchTree | Динамическое | Базовое BST |
| AVLTree | Динамическое | Самобалансирующееся |
| Treap | Динамическое | Декартово дерево |
| OptimalTree-A1 | Статическое | Алгоритм 1 |
| OptimalTree-A2 | Статическое | Алгоритм 2 |

**Метрики:**
- Время построения
- Время поиска (равномерное распределение запросов)
- Время поиска (Zipf-распределение запросов)
- Высота дерева
- Средневзвешенная высота (hср)

## Ожидаемые результаты

- **hср(Algo2) < hср(Algo1)** — Алгоритм 2 даёт лучшую структуру
- При Zipf-нагрузке ДОП может быть быстрее AVL
- При равномерной нагрузке AVL и ДОП сопоставимы

## Выполненные требования

- ✅ Алгоритм 1: сортировка по весам → BST
- ✅ Алгоритм 2: метод "центра тяжести"
- ✅ Сравнение времени построения
- ✅ Сравнение времени поиска
- ✅ Тест на dataset с тем же seed (12345)
- ✅ Включение в глобальный тест (BST, AVL, Treap, ДОП1, ДОП2)

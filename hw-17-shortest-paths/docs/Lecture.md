# Лекция 17. Алгоритмы поиска кратчайших путей в графах

## 1. Введение

Алгоритмы поиска кратчайших путей (Shortest Path Algorithms) — это фундаментальный класс алгоритмов на графах. Они используются в навигации, сетях, оптимизации маршрутов, анализе данных, задачах на поиск оптимальных последовательностей действий и многих других областях.

В рамках данной лекции рассматриваются четыре основных алгоритма:

- **Алгоритм Дейкстры** — поиск кратчайших путей от одной вершины в графе без отрицательных рёбер.
- **Алгоритм Беллмана–Форда** — корректно работает даже при наличии отрицательных рёбер.
- **Алгоритм Флойда–Уоршелла** — получение кратчайших путей между всеми парами вершин, работает с матрицей смежности.
- **Алгоритм Джонсона** — оптимальный способ поиска всех кратчайших путей в разреженных графах за счёт ре-взвешивания рёбер.

В лекции объясняются принципы работы, ограничения, примеры, преимущества и недостатки каждого метода.

---

## 2. Основные определения

### 2.1 Взвешенный граф

Граф **G = (V, E)**, где каждому ребру **(u, v)** сопоставлен вес **w(u, v)**.
Вес может быть:

- положительным
- нулевым
- отрицательным (кроме некоторых алгоритмов)

### 2.2 Кратчайший путь

Кратчайший путь от вершины **s** к вершине **v** — путь с минимальной суммой весов.

---

# 3. Алгоритм Дейкстры

## 3.1 Где используется

Алгоритм Дейкстры применяется для поиска кратчайших путей в графах со **взвешенными рёбрами**, где **все веса неотрицательны**.

Примеры:

- навигационные системы (Google Maps, GPS)
- маршрутизация в компьютерных сетях (OSPF)
- оптимизация графов состояний
- задачи планирования и логистики

---

## 3.2 Ограничения

- **Нельзя использовать**, если в графе есть отрицательные веса.
- Работает для **ориентированных и неориентированных** графов.

---

## 3.3 Основная идея

Идея алгоритма — жадная стратегия: каждый раз выбираем вершину с минимальной текущей оценкой расстояния и "фиксируем" её.

Ключевые структуры:

- массив расстояний `dist[]`
- множество посещённых вершин
- очередь с приоритетом (min-heap) — ускоренная версия

---

## 3.4 Псевдокод (приоритетная очередь)

```
function Dijkstra(graph, s):
    dist[v] = INF for all v
    parent[v] = NONE for all v
    dist[s] = 0
    pq.push( (0, s) )

    while pq is not empty:
        (d, u) = pq.extractMin()
        if d > dist[u]:
            continue

        for each edge (u, v) with weight w:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                pq.push( (dist[v], v) )

    return dist, parent
```

---

## 3.5 Восстановление пути

```
vector<int> restorePath(int v, vector<int>& parent):
    path = {}
    while v != -1:
        path.push_back(v)
        v = parent[v]
    reverse(path)
    return path
```

---

## 3.6 Важные детали реализации

- Для ускорения ‒ используем **матрицу смежности**, как в ДЗ-16.
- INF представляется большим числом (`INT_MAX / 2`).

---

## 3.7 Сложность

При использовании min-heap:

- **O((V + E) log V)**

При матрице смежности:

- **O(V² log V)**

---

## 3.8 Наглядный пример (из лекции)

Граф:

```
   7     9
A --- B --- C
|     | \   |
14    10  2  11
|     |    \ |
D ---- E ---- F
   9       6
```

Множество шагов алгоритма описывается аналогично лекционным материалам.

# 4. Алгоритм Беллмана–Форда

## 4.1 Где используется

Используется в графах с **отрицательными рёбрами**.

Основные применения:

- анализ денежных потоков (поиск арбитражных циклов)
- графы с временными ограничениями
- задачи, где могут быть штрафы или обратные бонусы

---

## 4.2 Ограничения

- Более медленный, чем Дейкстра.
- Но корректно обрабатывает отрицательные веса.
- Выявляет **отрицательные циклы**.

---

## 4.3 Идея алгоритма

На каждом шаге "расслабляем" все рёбра.

Если на **V-м** шаге расстояние можно улучшить — найден отрицательный цикл.

---

## 4.4 Псевдокод

```
function BellmanFord(graph, s):
    dist[v] = INF for all v
    parent[v] = NONE
    dist[s] = 0

    for i = 1 to V-1:
        for each edge (u, v, w):
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u

    for each edge (u, v, w):
        if dist[u] + w < dist[v]:
            report "negative cycle"

    return dist, parent
```

---

## 4.5 Сложность

- **O(V × E)**

Для матрицы смежности:  

- **O(V³)**

---

# 5. Алгоритм Флойда–Уоршелла

## 5.1 Где используется

Подходит для задач, где нужно найти кратчайшие пути между **всеми парами** вершин:

- вычисление дистанционных матриц
- анализ графов состояний
- маршрутизаторы (как основа некоторых протоколов)

---

## 5.2 Ограничения

- Работает и с отрицательными весами.
- Не работает при наличии отрицательных циклов.
- Требует **матрицу смежности**.

---

## 5.3 Идея

Динамическое программирование:  

```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

---

## 5.4 Псевдокод

```
for k in 1..V:
  for i in 1..V:
    for j in 1..V:
      if dist[i][k] + dist[k][j] < dist[i][j]:
          dist[i][j] = dist[i][k] + dist[k][j]
```

---

## 5.5 Сложность

- **O(V³)**

---

# 6. Алгоритм Джонсона

## 6.1 Где используется

Эффективен в **разреженных графах** (много вершин, мало рёбер) при необходимости найти кратчайшие пути между всеми вершинами.

Используется в системах, где графы очень большие:

- большие сети
- графы состояний

---

## 6.2 Идея

1. Добавить фиктивную вершину s.
2. Запустить Беллман–Форда → найти потенциалы h[v].
3. Перевзвесить рёбра:

```
w'(u, v) = w(u, v) + h[u] - h[v]
```

4. Запустить Дейкстру V раз.

---

## 6.3 Преимущества

- Работает в разреженных графах гораздо быстрее Флойда–Уоршелла.

---

## 6.4 Сложность

При использовании кучи и списка смежности:

- **O(V × (E log V))**

---

# 7. Сравнение алгоритмов

| Алгоритм      | Отриц. веса | Все пары | Сложность               |
| ------------- | ----------- | -------- | ----------------------- |
| Дейкстра      | ❌           | ❌        | Быстрый                 |
| Беллман–Форд  | ✔           | ❌        | Медленный               |
| Флойд–Уоршелл | ✔           | ✔        | O(V³)                   |
| Джонсон       | ✔           | ✔        | Отлично для разреженных |

# 8. Общие детали реализации

## 8.1 Формат графа (как в ДЗ-16)

Используем **матрицу смежности**:

- 0 (или INF) означает отсутствие ребра
- dist[i][j] содержит вес

## 8.2 Представление INF

```
const int INF = 1e9;
```

## 8.3 Восстановление пути

Используем `parent[]`.

## 8.4 Визуализация (как в ДЗ-16)

- таблицы расстояний
- списки шагов

# 9. Итоги

В лекции рассмотрены четыре основных алгоритма поиска кратчайших путей:

- Дейкстра для графов без отрицательных рёбер,
- Беллман–Форд — для графов с отрицательными рёбрами,
- Флойд–Уоршелл — для всех пар вершин,
- Джонсон — для больших разреженных графов.

# ДЗ-17: Алгоритмы поиска кратчайших путей

## 🧓 Объяснение алгоритмов простым языком ("Расскажи бабушке")

Представь, бабушка, что в нашем районе много деревень, и между ними есть дороги. На каждой дороге написано, сколько километров ехать. Мы хотим найти самый короткий путь.

---

### Алгоритм Дейкстры: "Как добраться до больницы?"

**Задача:** Найти кратчайший путь от нашей деревни до всех остальных.

**Как это делает бабушка:**

1. **Берём листок бумаги.** Пишем список всех деревень. Напротив своей ставим "0 км" — мы уже здесь! Напротив остальных пишем "?" — пока не знаем.

2. **Смотрим на дорожные указатели.** Из нашей деревни видим: "До Петровки — 5 км", "До Ивановки — 3 км". Записываем это в листок.

3. **Едем в ближайшую.** Ивановка ближе (3 км) — едем туда. Ставим галочку: "Ивановку проверили, точно 3 км".

4. **Из Ивановки смотрим дальше.** Отсюда до Сидоровки — 2 км. Значит, от дома до Сидоровки через Ивановку = 3 + 2 = 5 км. Записываем!

5. **Выбираем следующую ближайшую.** Теперь Петровка и Сидоровка обе по 5 км. Берём любую, проверяем её соседей...

6. **Повторяем**, пока не проверим все деревни.

**Почему это работает?**

Когда мы ставим галочку напротив деревни — мы уверены, что короче уже не будет. Ведь все оставшиеся деревни дальше, а значит любой путь через них будет длиннее!

**Простая аналогия:** Как волна от камня в пруду — сначала доходит до ближнего берега, потом до дальнего. Волна всегда идёт по кратчайшему пути.

---

### Алгоритм Флойда-Уоршелла: "Таблица расстояний для всего района"

**Задача:** Составить таблицу расстояний между ВСЕМИ парами деревень.

**Как это делает бабушка:**

1. **Рисуем таблицу.** По вертикали и горизонтали — названия всех деревень. В клеточках — расстояния по прямым дорогам. Если прямой дороги нет — ставим прочерк.

2. **Думаем про каждую деревню как про пересадку.** Берём Сидоровку и спрашиваем: "А вдруг от Петровки до Ивановки через Сидоровку короче?" Проверяем, если короче — исправляем в таблице.

3. **Проверяем все деревни как пересадки.** Сначала всё через Сидоровку, потом всё через Михайловку, потом через Борисовку... И так для каждой деревни района.

**Пример:**
- Напрямую от Петровки до Ивановки — 10 км
- Через Сидоровку: Петровка → Сидоровка (3 км) + Сидоровка → Ивановка (4 км) = 7 км
- Ага! 7 < 10, исправляем в таблице!

**Почему это работает?**

Когда мы проверили все деревни как возможные пересадки — мы точно нашли самый короткий путь. Даже если он идёт через три деревни — мы его найдём!

---

### Когда какой алгоритм использовать?

| Ситуация | Алгоритм | Почему |
|----------|----------|--------|
| "Как мне доехать до больницы?" | Дейкстра | Нужен путь от одной точки |
| "Составить справочник расстояний для всех" | Флойд-Уоршелл | Нужны все пары сразу |
| "Есть дороги с отрицательной длиной" | Флойд-Уоршелл | Дейкстра не умеет с отрицательными |

## Описание

Реализация двух алгоритмов поиска кратчайших путей:

1. **Алгоритм Дейкстры** — от одной вершины до всех остальных (неотрицательные веса)
2. **Алгоритм Флойда-Уоршелла** — между всеми парами вершин (поддерживает отрицательные веса)

## Алгоритм Дейкстры

### Идея

Жадный алгоритм: на каждом шаге выбираем непосещённую вершину с минимальным расстоянием от источника и "релаксируем" все её соседей.

### Псевдокод

```
function Dijkstra(graph, source):
    dist[v] = INF для всех v
    dist[source] = 0
    pq = PriorityQueue()
    pq.push(0, source)
    
    while pq не пуста:
        u = pq.extractMin()
        if visited[u]: continue
        visited[u] = true
        
        for each сосед v вершины u:
            if dist[u] + weight(u,v) < dist[v]:
                dist[v] = dist[u] + weight(u,v)
                parent[v] = u
                pq.push(dist[v], v)
    
    return dist, parent
```

### Сложность

| Представление | Время | Память |
|---------------|-------|--------|
| Матрица смежности + heap | O(V² log V) | O(V²) |
| Список смежности + heap | O((V+E) log V) | O(V+E) |

### Ограничения

- Работает **только** с неотрицательными весами
- Для отрицательных весов используйте Беллмана-Форда

## API

### Использование

```cpp
#include "Dijkstra.hpp"
#include "GraphMatrix.hpp"

// Создание графа
GraphMatrix g(5);
g.addUndirectedEdge(0, 1, 7);
g.addUndirectedEdge(1, 2, 3);
// ...

// Запуск алгоритма
auto result = Dijkstra::execute(g, 0);  // от вершины 0

// Получение результатов
int dist = result.dist.get(2);          // расстояние до вершины 2
auto path = result.getPath(2);          // путь до вершины 2

// Дерево кратчайших путей (рёбра)
for (int i = 0; i < result.edges.size(); ++i) {
    Edge e = result.edges.get(i);
    std::cout << e.v1 << " -> " << e.v2 << "\n";
}
```

### Формат результата

Согласно заданию, результат представлен в виде `Edge[] edges`:

```cpp
struct Edge {
    int v1;  // первая вершина
    int v2;  // вторая вершина
};

struct ShortestPathResult {
    FactorArray<Edge> edges;    // рёбра дерева кратчайших путей
    FactorArray<int> dist;      // расстояния
    FactorArray<int> parent;    // родители (для восстановления пути)
    int sourceVertex;           // исходная вершина
    bool allReachable;          // все ли вершины достижимы
};
```

## Алгоритм Флойда-Уоршелла

### Идея

Находит кратчайшие пути между **всеми парами** вершин за один запуск. Работает с отрицательными весами и обнаруживает отрицательные циклы.

### Псевдокод

```
function FloydWarshall(graph):
    dist = копия матрицы весов графа
    
    for k от 0 до V-1:           # k — промежуточная вершина
        for i от 0 до V-1:
            for j от 0 до V-1:
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    
    return dist
```

### Сложность

| Время | Память |
|-------|--------|
| O(V³) | O(V²) |

### Преимущества и недостатки

| ✅ Преимущества | ❌ Недостатки |
|----------------|---------------|
| Простейшая реализация | Кубическая сложность |
| Все пары за один запуск | Не подходит для больших графов |
| Работает с отрицательными весами | O(V²) памяти |
| Обнаруживает отрицательные циклы | |

### API

```cpp
#include "FloydWarshall.hpp"
#include "GraphMatrix.hpp"

// Создание графа
GraphMatrix g(4);
g.addDirectedEdge(0, 1, 1);
g.addDirectedEdge(1, 2, -3);  // Отрицательный вес — OK!
// ...

// Запуск алгоритма
auto result = FloydWarshall::execute(g);

// Расстояние между любыми двумя вершинами
int d = result.getDistance(0, 3);

// Путь между вершинами
auto path = result.getPath(0, 3);

// Проверка на отрицательный цикл
if (result.hasNegativeCycle) {
    std::cout << "Граф содержит отрицательный цикл!\n";
}
```

## Сравнение алгоритмов

| Алгоритм | Сложность | Отриц. веса | Все пары | Когда использовать |
|----------|-----------|-------------|----------|-------------------|
| **Дейкстра** | O((V+E) log V) | ❌ | ❌ | Один источник, неотриц. веса |
| **Флойд-Уоршелл** | O(V³) | ✅ | ✅ | Маленькие графы, все пары |
| Беллман-Форд | O(V×E) | ✅ | ❌ | Один источник, отриц. веса |
| Джонсон | O(V×E log V) | ✅ | ✅ | Большие разреженные графы |

## Примеры

### Граф из лекции

```
       7         9
 [0] ----- [1] ----- [2]
  |         |\        |
  14       10  2      11
  |         |    \    |
 [3] ----- [4] ----- [5]
       9         6
```

Кратчайшие расстояния от вершины [0]:
- [0] → [0]: 0
- [0] → [1]: 7
- [0] → [2]: 16 ([0]→[1]→[2])
- [0] → [3]: 14
- [0] → [4]: 9 ([0]→[1]→[4])
- [0] → [5]: 15 ([0]→[1]→[4]→[5])

## Тесты

### Дейкстра
- Базовые случаи (пустой граф, одна вершина)
- Простые графы (треугольник, линия, звезда)
- Ориентированные и неориентированные графы
- Несвязные графы
- Восстановление пути
- Выбор кратчайшего пути из нескольких
- Отрицательные веса (проверка исключения)
- Большие графы

### Флойд-Уоршелл
- Базовые случаи
- Ориентированные и неориентированные графы
- Восстановление пути между любыми вершинами
- Отрицательные веса (должен корректно работать)
- Обнаружение отрицательных циклов
- Сравнение результатов с Дейкстрой

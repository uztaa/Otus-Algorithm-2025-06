## Основные Концепции

### Что такое Хеш-таблица?

**Хеш-таблица** — это структура данных, которая реализует ассоциативный массив (отображение ключ-значение). Основная идея заключается в использовании **хеш-функции** для преобразования ключа в индекс массива, где хранится значение.

### Хеш-функция

**Хеш-функция** `h(k)` принимает ключ `k` и возвращает индекс в таблице размера `m`:

$$h(k) : U \to [0, m-1]$$

где `U` — универсум всех возможных ключей.

**Характеристики хорошей хеш-функции:**
- Детерминированность: один и тот же ключ всегда дает один и тот же хеш
- Равномерное распределение: хеши распределены равномерно по таблице
- Быстрое вычисление: O(1) или близко к этому
- Минимизация коллизий

### Коллизия

**Коллизия** происходит, когда двe разные ключи `k₁` и `k₂` отображаются на один и тот же индекс: `h(k₁) = h(k₂)`.

По принципу Дирихле, коллизии неизбежны, если количество ключей больше размера таблицы.

### Коэффициент Заполнения (Load Factor)

**Коэффициент заполнения** α определяется как:

$$\alpha = \frac{n}{m}$$

где:
- `n` — количество элементов в таблице
- `m` — размер таблицы

Этот параметр критически влияет на производительность хеш-таблицы. Обычно поддерживается α ≤ 0.75 для открытой адресации и α ≤ 1.0 для метода цепочек.

## Метод Цепочек (Chaining)

### Принцип Работы

В методе цепочек каждый элемент таблицы является **связным списком** (цепочкой) элементов, которые хешируются в один и тот же индекс.

```
Таблица с цепочками (size = 5):

Index:  0  1  2  3  4
        |  |  |  |  |
        v  v  v  v  v
        □  □  □  □  □
        |  |  |  |
        v  v  |  v
        50 15 v  85
        |  |  |  |
        v  v  v  v
        70 12 11 93
            |
            v
            76
```

### Операции

**Вставка (Insert):**
1. Вычислить хеш ключа: `index = h(key) % m`
2. Перейти в соответствующий список (цепочку)
3. Проверить, есть ли уже такой ключ
4. Если нет, вставить в начало списка (O(1))

```cpp
void insert(int key, int value) {
    int index = hash(key) % size;
    List* chain = table[index];
    
    // Проверяем на дубликаты
    for (auto node : chain) {
        if (node->key == key) {
            node->value = value;  // Обновляем значение
            return;
        }
    }
    
    // Вставляем в начало цепочки
    chain.pushFront({key, value});
}
```

**Поиск (Search):**
1. Вычислить хеш ключа
2. Пройти по цепочке в поиске ключа
3. Вернуть значение, если найдено

```cpp
bool search(int key, int& value) {
    int index = hash(key) % size;
    
    for (auto node : table[index]) {
        if (node->key == key) {
            value = node->value;
            return true;
        }
    }
    return false;
}
```

**Удаление (Remove):**
1. Вычислить хеш ключа
2. Найти и удалить элемент из цепочки

```cpp
void remove(int key) {
    int index = hash(key) % size;
    table[index].remove(key);
}
```

### Анализ Сложности (Chaining)

- **Успешный поиск:** O(1 + α) в среднем
- **Неудачный поиск:** O(α) в среднем
- **Вставка:** O(1) в среднем
- **Удаление:** O(1) в среднем (если элемент найден)

При α ≤ 1, все операции работают в O(1) в среднем.

**Худший случай:** O(n) если все элементы хешируются в один индекс

### Преимущества Chaining
- Простая реализация
- Удаление работает эффективно
- Хорошая производительность при α > 1

### Недостатки Chaining
- Требует дополнительной памяти для связных списков
- Плохая локальность кеша (при использовании указателей)
- При больших α может быть медленнее, чем открытая адресация

## Открытая Адресация (Open Addressing)

### Принцип Работы

В методе открытой адресации **все элементы хранятся в самой таблице**. Когда происходит коллизия, алгоритм ищет другую пустую ячейку в таблице, используя **последовательность пробирования** (probe sequence).

```
Открытая адресация (Linear Probing):

Исходно:        После вставки 50:    После вставки 15:
Index   Value   Index   Value        Index   Value
0       [nil]   0       50           0       50
1       [nil]   1       [nil]        1       15
2       [nil]   2       [nil]        2       [nil]
3       [nil]   3       [nil]        3       [nil]
4       [nil]   4       [nil]        4       [nil]
```

### Функция Пробирования

**Функция пробирования** определяет последовательность индексов для проверки:

$$h(k, i) = (h'(k) + f(i)) \mod m$$

где:
- `h'(k)` — основная хеш-функция
- `f(i)` — функция пробирования
- `i` — номер попытки (0, 1, 2, ...)

### Типы Пробирования

#### 1. Линейное Пробирование (Linear Probing)

$$h(k, i) = (h'(k) + i) \mod m$$

**Последовательность:** `h'(k), h'(k)+1, h'(k)+2, h'(k)+3, ...`

**Пример:** с хеш-функцией `hash(x) = x % 5` для вставки 50, 70, 76, 85, 93:
- 50 % 5 = 0 → индекс 0
- 70 % 5 = 0 (коллизия) → попытаемся 1 → индекс 1
- 76 % 5 = 1 (коллизия) → попытаемся 2 → индекс 2
- 85 % 5 = 0 (коллизия) → попытаемся 1 (занято) → попытаемся 2 (занято) → попытаемся 3 → индекс 3
- 93 % 5 = 3 (коллизия) → попытаемся 4 → индекс 4

```
Index   0    1    2    3    4
Value   50   70   76   85   93
```

**Проблема:**
- **Первичная кластеризация** (Primary Clustering): длинные последовательности заполненных ячеек замедляют поиск

#### 2. Квадратичное Пробирование (Quadratic Probing)

$$h(k, i) = (h'(k) + c_1 \cdot i + c_2 \cdot i^2) \mod m$$

**Типичный вариант:** с `c₁ = 1, c₂ = 1`

$$h(k, i) = (h'(k) + i + i^2) \mod m$$

**Последовательность:** `h'(k), h'(k)+1, h'(k)+4, h'(k)+9, h'(k)+16, ...`

```cpp
int index = hash(key);
for (int i = 0; i < size; i++) {
    int probeIndex = (index + i + i*i) % size;
    if (table[probeIndex] == EMPTY) {
        table[probeIndex] = key;
        return;
    }
}
```

**Преимущества над линейным пробированием:**
- Уменьшает первичную кластеризацию
- Хорошая производительность на практике

**Недостатки:**
- Может не посетить все ячейки таблицы (зависит от m)
- Вторичная кластеризация (Secondary Clustering)

**Гарантия полного обхода:** если `m` — степень двойки, то можно обеспечить посещение всех ячеек, использовав:

$$h(k, i) = (h'(k) + \frac{i(i+1)}{2}) \mod m$$

#### 3. Двойное Хеширование (Double Hashing)

$$h(k, i) = (h_1(k) + i \cdot h_2(k)) \mod m$$

**Пример:**
- `h₁(k) = k % m` — первичная хеш-функция
- `h₂(k) = 1 + (k % (m-1))` — вторичная хеш-функция

**Последовательность:** `h₁(k), h₁(k) + h₂(k), h₁(k) + 2·h₂(k), ...`

```cpp
int hash1(int key) {
    return key % size;
}

int hash2(int key) {
    return 1 + (key % (size - 1));
}

int index = hash1(key);
int step = hash2(key);
for (int i = 0; i < size; i++) {
    int probeIndex = (index + i * step) % size;
    if (table[probeIndex] == EMPTY) {
        table[probeIndex] = key;
        return;
    }
}
```

**Преимущества:**
- Минимизирует кластеризацию
- Использует две независимые хеш-функции
- Гарантирует посещение всех ячеек, если `gcd(h₂(k), m) = 1`

**Недостатки:**
- Требует вычисления двух хеш-функций
- Более сложная реализация

### Операции с Открытой Адресацией

**Вставка:**
```cpp
void insert(int key) {
    for (int i = 0; i < size; i++) {
        int index = (hash(key) + probe(i)) % size;
        if (table[index] == EMPTY || table[index] == DELETED) {
            table[index] = key;
            count++;
            return;
        }
    }
    // Таблица полна или нужен rehash
}
```

**Поиск:**
```cpp
bool search(int key) {
    for (int i = 0; i < size; i++) {
        int index = (hash(key) + probe(i)) % size;
        if (table[index] == EMPTY)
            return false;  // Ключа нет
        if (table[index] == key)
            return true;   // Ключ найден
        // Если DELETED, продолжаем поиск
    }
    return false;
}
```

**Удаление:** см. раздел "Ленивое удаление"

## Специальные Техники

### Ленивое Удаление (Lazy Deletion)

При удалении элемента из таблицы с открытой адресацией нельзя просто оставить пустую ячейку, так как это нарушит последовательность пробирования для других элементов.

**Решение:** вместо физического удаления отмечать ячейку как "DELETED".

```cpp
enum CellState {
    EMPTY,      // Никогда не была занята
    ACTIVE,     // Содержит активный элемент
    DELETED     // Была занята, но элемент удален
};

struct Cell {
    int key;
    CellState state;
};

Cell table[TABLE_SIZE];

void remove(int key) {
    for (int i = 0; i < size; i++) {
        int index = (hash(key) + probe(i)) % size;
        if (table[index].state == EMPTY)
            return;  // Ключа нет
        if (table[index].state == ACTIVE && table[index].key == key) {
            table[index].state = DELETED;  // Отмечаем как удаленный
            count--;
            return;
        }
    }
}

bool search(int key) {
    for (int i = 0; i < size; i++) {
        int index = (hash(key) + probe(i)) % size;
        if (table[index].state == EMPTY)
            return false;
        if (table[index].state == ACTIVE && table[index].key == key)
            return true;
        // Если DELETED, продолжаем поиск
    }
    return false;
}

void insert(int key) {
    for (int i = 0; i < size; i++) {
        int index = (hash(key) + probe(i)) % size;
        if (table[index].state != ACTIVE) {  // EMPTY или DELETED
            table[index].key = key;
            table[index].state = ACTIVE;
            count++;
            return;
        }
    }
}
```

### Проблемы Ленивого Удаления

1. **Фрагментация:** Накопление DELETED ячеек замедляет поиск и вставку
2. **Решение:** Периодический rehash при достижении порога DELETED ячеек

```cpp
void rehash() {
    Cell* oldTable = table;
    int oldSize = size;
    
    size = nextPrime(size * 2);
    table = new Cell[size];
    
    for (int i = 0; i < size; i++) {
        table[i].state = EMPTY;
    }
    
    count = 0;
    for (int i = 0; i < oldSize; i++) {
        if (oldTable[i].state == ACTIVE) {
            insert(oldTable[i].key);
        }
    }
    
    delete[] oldTable;
}
```

### Оптимальное Удаление (для открытой адресации)

Можно реализовать удаление без DELETED пометок, перемещая элементы после удаления. Однако это требует более сложной логики.

## Анализ Сложности

### Метод Цепочек

| Операция | Средний случай | Худший случай |
|----------|---|---|
| Поиск | O(1 + α) | O(n) |
| Вставка | O(1) | O(n) |
| Удаление | O(1) | O(n) |

где α = n/m (коэффициент заполнения)

### Открытая Адресация (без ленивого удаления)

| Операция | Средний случай | Худший случай |
|----------|---|---|
| Успешный поиск | O(-ln(1-α)/α) ≈ O(1) при α < 0.7 | O(n) |
| Неудачный поиск | O(1/(1-α)) | O(n) |
| Вставка | O(1/(1-α)) | O(n) |
| Удаление | O(-ln(1-α)/α) | O(n) |

### Открытая Адресация (с ленивым удалением)

При достаточно низкой доле DELETED ячеек (например, < 20%), производительность близка к обычной открытой адресации.

## Практические Соображения

### Выбор между Chaining и Open Addressing

**Используйте Chaining если:**
- Нужно обрабатывать удаления часто
- Load factor часто превышает 1.0
- Предпочитаете простоту реализации
- Есть много коллизий

**Используйте Open Addressing если:**
- Нужна лучшая локальность кеша
- Load factor обычно < 0.8
- Мало удалений
- Нужно минимизировать дополнительную память

### Выбор Хеш-Функции

**Хорошие хеш-функции:**

1. **Для целых чисел:** `h(k) = k mod p`, где p — простое число близко к m
2. **Для строк:** полиномиальное хеширование
   ```
   h(s) = (s[0]*31^0 + s[1]*31^1 + ... + s[n-1]*31^(n-1)) mod m
   ```
3. **Криптографические:** MurmurHash, xxHash (для некритичных случаев)

### Размер Таблицы

- Выбирайте **простые числа** для размера m (снижает коллизии)
- Для открытой адресации можно использовать **степени двойки** (быстрее вычислять mod)
- Увеличивайте размер при α > 0.75-0.8

### Когда Делать Rehash

```cpp
if (count > 0.75 * size) {
    rehash();  // Увеличить размер таблицы
}
```

## Резюме

**Хеш-таблицы** — один из самых важных структур данных, обеспечивающих O(1) операции поиска в среднем.

**Два основных подхода:**
1. **Chaining** — простой и гибкий, хорошо для частых удалений
2. **Open Addressing** — более эффективен по памяти, лучше локальность кеша

**Специальные техники:**
- **Ленивое удаление** — помечаем удаленные ячейки, требует периодического rehash
- **Квадратичное пробирование** — уменьшает кластеризацию в open addressing
- **Двойное хеширование** — еще лучше борется с кластеризацией

**Ключ к производительности:**
- Хорошая хеш-функция
- Поддержание низкого коэффициента заполнения
- Правильный выбор между chaining и open addressing

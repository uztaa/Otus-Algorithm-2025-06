# Алгоритм Демукрона: Подробный разбор

## Часть 1: Что такое топологическая сортировка и зачем она нужна

Представь, что ты планируешь ремонт квартиры. Есть задачи: "снять старые обои", 
"выровнять стены", "поклеить новые обои", "установить розетки". Очевидно, что нельзя 
клеить обои до выравнивания стен, а розетки нужно проложить до поклейки. 
Это **граф зависимостей**, где рёбра показывают "что должно быть сделано раньше".

**Топологическая сортировка** — это упорядочивание таких задач в линейную 
последовательность, где каждая задача идёт после всех своих зависимостей.

## Часть 2: Чем Демукрон отличается от обычной топологической сортировки

Обычные алгоритмы (Кан, Тарьян) выдают **линейный список** вершин. 
Демукрон выдаёт **разбиение на уровни** — группы задач, которые можно выполнять **параллельно**.

Возвращаясь к ремонту:

```
Уровень 0: [снять обои, демонтировать старую проводку]  ← можно делать параллельно
Уровень 1: [выровнять стены, проложить новую проводку]  ← можно делать параллельно  
Уровень 2: [установить розетки, поклеить обои]          ← можно делать параллельно
Уровень 3: [расставить мебель]
```

Это важно для планирования работ, компиляции (какие модули можно собирать параллельно), 
распределённых вычислений.

## Часть 3: Ключевые понятия

**Полустепень захода (in-degree)** вершины v — количество рёбер, **входящих** в v. 
Это число "зависимостей" — сколько задач должно быть выполнено до v.

```
    A ──→ C ←── B
          ↓
          D
          
Полустепени захода:
- A: 0 (никто не указывает на A)
- B: 0  
- C: 2 (на C указывают A и B)
- D: 1 (на D указывает C)
```

**Уровень вершины** — максимальная длина пути от любого "входа" графа 
(вершины с нулевой полустепенью захода) до данной вершины. 
Иначе говоря: "на сколько шагов ты отстоишь от старта?"

## Часть 4: Идея алгоритма Демукрона

Алгоритм работает "послойно", как очистка луковицы:

1. **Найди все вершины с полустепенью захода = 0.** 
   Это "входы" графа, задачи без зависимостей. Они формируют **уровень 0**.

2. **"Удали" эти вершины и все исходящие из них рёбра.** 
   После удаления у некоторых вершин полустепень захода уменьшится и станет равна 0.

3. **Новые вершины с нулевой полустепенью** формируют **уровень 1**.

4. **Повторяй**, пока не обработаешь все вершины.

Если на каком-то шаге нет вершин с нулевой полустепенью, а граф ещё не пуст — 
**в графе есть цикл**, топологическая сортировка невозможна.

## Часть 5: Пошаговый пример

Возьмём простой граф:

```
    0 ──→ 2 ──→ 4
    ↓     ↑
    1 ────┘
    
Рёбра: 0→1, 0→2, 1→2, 2→4
```

**Начальные полустепени захода:**

```
Вершина:  0   1   2   3   4
Захода:   0   1   2   0   1
          ↑               ↑
        вход            вход (изолированная)
```

Вершины 0 и 3 имеют полустепень 0. Но 3 — изолированная вершина (нет рёбер), 
её ставим на уровень 0.

**Уровень 0:** {0} (и {3} если считаем изолированные)

**Удаляем 0**, вычитаем рёбра 0→1, 0→2:

```
Вершина:  0   1   2   4
Захода:   -   0   1   1
```

**Уровень 1:** {1}

**Удаляем 1**, вычитаем ребро 1→2:

```
Вершина:  0   1   2   4  
Захода:   -   -   0   1
```

**Уровень 2:** {2}

**Удаляем 2**, вычитаем ребро 2→4:

```
Вершина:  0   1   2   4
Захода:   -   -   -   0
```

**Уровень 3:** {4}

**Результат:**

```
level[0] = {0, 3}
level[1] = {1}
level[2] = {2}
level[3] = {4}
```

## Часть 6: Псевдокод алгоритма

```
Демукрон(граф, количество_вершин):
    // Шаг 1: Вычисляем начальные полустепени захода
    M[v] = 0 для всех v
    для каждого ребра (u, v) в графе:
        M[v]++
    
    обработано = пустое множество
    уровень = 0
    
    // Шаг 2: Основной цикл
    пока размер(обработано) < количество_вершин:
        // Находим вершины с нулевой полустепенью
        zero = {v : M[v] == 0 и v не в обработано}
        
        если zero пусто:
            вернуть ОШИБКА: "Граф содержит цикл"
        
        // Добавляем на текущий уровень
        результат[уровень] = zero
        
        // "Удаляем" вершины
        для каждой v из zero:
            добавить v в обработано
            для каждого соседа w вершины v:
                M[w]--
        
        уровень++
    
    вернуть результат
```

## Часть 7: Сложность алгоритма

- **Время**: O(V + E), где V — количество вершин, E — количество рёбер
  - Вычисление начальных полустепеней: O(E)
  - Основной цикл: каждая вершина и каждое ребро обрабатываются ровно один раз

- **Память**: O(V) для массива полустепеней и множества обработанных вершин

## Часть 8: Практическое применение

1. **Сборка проектов**: определение порядка компиляции модулей с зависимостями
2. **Планирование задач**: какие задачи можно выполнять параллельно
3. **Курсы обучения**: в каком порядке проходить курсы с пререквизитами
4. **Электронные таблицы**: порядок вычисления ячеек с формулами

## Часть 9: Связь с другими алгоритмами

- **Алгоритм Кана**: похож на Демукрона, но выдаёт линейный порядок вместо уровней
- **Алгоритм Тарьяна**: использует DFS с 3-цветной раскраской, также для топологической сортировки
- **Поиск мостов/точек сочленения**: другие алгоритмы на графах, использующие DFS

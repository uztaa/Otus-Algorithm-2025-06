### JUNIOR уровень (2 байта)

#### SEL1. Selection Sort (+1 байт)
✅ **Выполнено**

**Описание алгоритма:**
Selection Sort (сортировка выбором) - это простой алгоритм сортировки, который работает следующим образом:
1. Находит минимальный элемент в неотсортированной части массива
2. Меняет его местами с первым элементом неотсортированной части
3. Увеличивает границу отсортированной части на один элемент
4. Повторяет шаги 1-3 до полной сортировки массива

**Характеристики:**
- **Временная сложность**: O(n²) во всех случаях (best, average, worst)
- **Пространственная сложность**: O(1) - сортировка на месте
- **Стабильность**: Нестабильная (можно сделать стабильной с модификациями)
- **Количество сравнений**: Всегда n(n-1)/2
- **Количество обменов**: Максимум n-1

**Реализация:**
```cpp
void SelectionSorter::sort(std::vector<Record>& arr) {
    notify(StartEvent(name));
    
    size_t n = arr.size();
    for (size_t i = 0; i < n - 1; ++i) {
        size_t minIdx = findMinIndex(arr, i, n);
        if (minIdx != i) {
            notify(SwapEvent(name, i, minIdx));
            std::swap(arr[i], arr[minIdx]);
        }
    }
    
    notify(FinishEvent(name));
}
```

**Преимущества:**
- Простота реализации
- Работает на месте (не требует дополнительной памяти)
- Минимальное количество обменов (полезно, когда обмен - дорогая операция)

**Недостатки:**
- Квадратичная сложность O(n²)
- Неэффективен для больших массивов
- Нестабильная сортировка

#### HIP1. Heap Sort (+1 байт)
✅ **Выполнено**

**Описание алгоритма:**
Heap Sort (пирамидальная сортировка) - алгоритм сортировки, использующий структуру данных "куча" (heap):
1. Строит max-heap из входного массива
2. Извлекает максимальный элемент (корень кучи) и помещает в конец
3. Уменьшает размер кучи на 1
4. Восстанавливает свойство кучи (heapify)
5. Повторяет шаги 2-4 до полной сортировки

**Характеристики:**
- **Временная сложность**: O(n log n) во всех случаях (best, average, worst)
- **Пространственная сложность**: O(1) - сортировка на месте
- **Стабильность**: Нестабильная
- **Предсказуемость**: Всегда O(n log n), нет худшего случая

**Реализация:**
```cpp
void HeapSorter::sort(std::vector<Record>& arr) {
    notify(StartEvent(name));
    
    size_t n = arr.size();
    buildMaxHeap(arr);
    
    for (size_t i = n - 1; i > 0; --i) {
        notify(SwapEvent(name, 0, i));
        std::swap(arr[0], arr[i]);
        heapify(arr, i, 0);
    }
    
    notify(FinishEvent(name));
}

void HeapSorter::heapify(std::vector<Record>& arr, size_t n, size_t i) {
    size_t largest = i;
    size_t left = 2 * i + 1;
    size_t right = 2 * i + 2;
    
    if (left < n && arr[left].getKey() > arr[largest].getKey())
        largest = left;
    
    if (right < n && arr[right].getKey() > arr[largest].getKey())
        largest = right;
    
    if (largest != i) {
        std::swap(arr[i], arr[largest]);
        heapify(arr, n, largest);
    }
}
```

**Преимущества:**
- Гарантированная сложность O(n log n)
- Работает на месте (не требует дополнительной памяти)
- Эффективен для больших массивов
- Нет худшего случая (в отличие от QuickSort)

**Недостатки:**
- Нестабильная сортировка
- Практически медленнее QuickSort (из-за плохой локальности данных)
- Более сложная реализация, чем простые O(n²) алгоритмы

## Архитектурные решения

### 1. Использование базового класса BaseSorter
Оба алгоритма наследуются от `BaseSorter`, который реализует интерфейс `Sortable`:
```cpp
class SelectionSorter : public BaseSorter { ... };
class HeapSorter : public BaseSorter { ... };
```

**Преимущества:**
- Единообразный интерфейс для всех алгоритмов
- Возможность полиморфного использования
- Автоматическая поддержка событийной модели

### 2. Событийная модель (Observer Pattern)
Использование событий для мониторинга процесса сортировки:
- `StartEvent` - начало сортировки
- `CompareEvent` - сравнение элементов
- `SwapEvent` - обмен элементов  
- `FinishEvent` - завершение сортировки

**Применение:**
- `DurationSortListener` - измерение времени выполнения
- `LoggingSortListener` - логирование операций (для отладки)
- Возможность добавления своих слушателей без изменения кода сортировщиков

### 3. Разделение на .hpp и .cpp
Следование best practices:
- Объявления в заголовочных файлах (`include/`)
- Реализация в исходных файлах (`src/`)
- Документация в стиле Doxygen

### 4. Именование методов
Использование `camelCase` согласно требованиям:
```cpp
void sort(std::vector<Record>& arr);
size_t findMinIndex(const std::vector<Record>& arr, size_t start, size_t end);
void heapify(std::vector<Record>& arr, size_t n, size_t i);
void buildMaxHeap(std::vector<Record>& arr);
```

## Сравнительный анализ алгоритмов

### Selection Sort
**Когда использовать:**
- Маленькие массивы (n < 100)
- Когда обмен элементов - дорогая операция (минимизация обменов)
- Когда важна простота реализации
- Обучающие цели

**Когда НЕ использовать:**
- Большие массивы (n > 1000)
- Когда нужна производительность
- Когда важна стабильность сортировки

### Heap Sort
**Когда использовать:**
- Большие массивы
- Когда нужна гарантированная O(n log n) сложность
- Когда критична память (сортировка на месте)
- Когда неприемлем худший случай O(n²) (как у QuickSort)

**Когда НЕ использовать:**
- Когда важна стабильность
- Когда критична скорость (QuickSort обычно быстрее на практике)
- Маленькие массивы (overhead от построения кучи)

### Сравнительная таблица

| Характеристика | Selection Sort | Heap Sort |
|----------------|----------------|-----------|
| Лучший случай | O(n²) | O(n log n) |
| Средний случай | O(n²) | O(n log n) |
| Худший случай | O(n²) | O(n log n) |
| Память | O(1) | O(1) |
| Стабильность | Нет | Нет |
| Обменов | O(n) | O(n log n) |
| Сравнений | O(n²) | O(n log n) |

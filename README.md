# Алгоритмы и структуры данных (C++)

Образовательный проект, включающий реализацию фундаментальных алгоритмов и структур данных на C++17. Работы представляют собой выполненные домашние задания.

## Технологический стек

| Компонент | Версия/Инструмент |
|-----------|-------------------|
| Язык | C++17 |
| Сборка | CMake 3.16+ |
| Генератор | Ninja |
| Тестирование | GoogleTest (gtest) |
| Компилятор | GCC 9+ / Clang 10+ |

## Структура проекта


Проект организован как набор независимых подпроектов (homeworks) и общих библиотек, объединенных корневым CMakeLists.txt.

```bash
.
├── common/                 # Общая библиотека (утилиты, логгеры, базовые интерфейсы)
├── cache-lib/              # Библиотека итогового проекта (в будущем — отдельный репо)
├── hw-01-xxx/              # Домашнее задание 1 (отдельный CMake подпроект)
├── hw-02-xxx/              # Домашнее задание 2 (отдельный CMake подпроект)
├── ...                     # Последующие домашние задания
├── external/               # Внешние зависимости (GoogleTest и др.)
├── CMakeLists.txt          # Корневой сценарий сборки
└── README.md
```

Основные модули

    common — содержит переиспользуемый код, необходимый во всех домашних заданиях (например, замеры времени, кастомные аллокаторы или вспомогательные макросы).

    cache-lib — ключевой модуль курсового проекта, реализующий механизмы кэширования. Разрабатывается с прицелом на выделение в самостоятельную библиотеку.

    hw-XX-name — изолированные решения конкретных задач курса (сортировки, деревья, графы). Каждая папка содержит свои исходники и тесты.


## Подход к разработке

### Кодовый стиль

- **Именование**: CamelCase для классов и функций, snake_case для переменных и функций-членов
- **Константность**: Используются `const` и `constexpr` где возможно
- **STL**: Предпочтение стандартным контейнерам (`std::vector`, `std::map`, `std::set`)
- **Исключения**: Минимальное использование, с явной обработкой ошибок

### Принципы реализации

1. **Чистота и читаемость** — код должен быть понятен без излишних комментариев
2. **Производительность** — анализ сложности O(n) и S(n) явно указан в комментариях
3. **Тестируемость** — каждый алгоритм покрыт unit-тестами
4. **Модульность** — отделение интерфейса от реализации

## Сборка

### Требования

- CMake 3.16+
- GCC/Clang с поддержкой C++17
- Ninja (опционально, но рекомендуется)

### Компиляция

```bash
mkdir build && cd build
cmake .. -G Ninja
ninja
```

Для сборки с тестами:
```bash
cmake .. -DBUILD_TESTS=ON -G Ninja
ninja
```

## Тестирование

Запуск всех тестов:
```bash
ctest --output-on-failure
```

Запуск тестов конкретного модуля:
```bash
ctest -R sorting --output-on-failure
```

Запуск с фильтром Google Test:
```bash
./tests/alg_tests --gtest_filter="SortingTest.*"
```

## Содержание

### Реализованные алгоритмы и структуры

- **Сортировка**: QuickSort, MergeSort, HeapSort, InsertionSort, BubbleSort
- **Поиск**: BinarySearch, LinearSearch
- **Структуры данных**: LinkedList, BinarySearchTree, AVLTree, HashTable, Graph (DFS/BFS)
- **Динамическое программирование**: LCS, Knapsack

Подробное описание каждого компонента см. в соответствующих файлах реализации.

## Примечания

- Все реализации содержат анализ временной и пространственной сложности
- Для большинства алгоритмов предусмотрены альтернативные версии (итеративная/рекурсивная)
- Тесты генерируют как малые (edge cases), так и большие наборы данных для проверки производительности

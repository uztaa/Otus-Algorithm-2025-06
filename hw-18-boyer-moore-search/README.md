## Реализованные алгоритмы

### 1. NaiveSearch — Наивный алгоритм (полный перебор)

- **Сложность препроцессинга:** O(1)
- **Сложность поиска:** O(n·m) в худшем случае
- **Память:** O(1)
- **Направление сравнения:** слева направо
- **Сдвиг при несовпадении:** +1

### 2. BMHSearch — Алгоритм Бойера-Мура-Хорспула

- **Сложность препроцессинга:** O(m + σ)
- **Сложность поиска:** O(n/m) лучший, O(n·m) худший
- **Память:** O(σ) = O(256)
- **Направление сравнения:** справа налево
- **Правило сдвига:** Bad Character Rule (по последнему символу окна)

### 3. BoyerMooreSearch — Полный алгоритм Бойера-Мура

- **Сложность препроцессинга:** O(m + σ)
- **Сложность поиска:** O(n/m) лучший, O(n+m) худший
- **Память:** O(m + σ)
- **Направление сравнения:** справа налево
- **Правила сдвига:** 
  - Bad Character Rule (по символу несовпадения)
  - Good Suffix Rule (по совпавшему суффиксу)
  - Выбирается максимальный из двух сдвигов

## Выводы

1. **Naive** — простейший алгоритм, неэффективен на повторяющихся данных. 
   На худшем случае (`AAAA...AB` в `AAAA...A`) показывает квадратичное время.

2. **BMH** — значительное ускорение за счёт пропуска позиций по Bad Character Rule.
   Особенно эффективен, когда символы образца не встречаются в тексте.
   Простая реализация, рекомендуется для большинства задач.

3. **Boyer-Moore** — дополнительное ускорение за счёт Good Suffix Rule.
   Наиболее эффективен на длинных образцах с повторяющимися подстроками.
   Гарантирует линейное время в худшем случае (с правилом Галиля).

**Рекомендации по выбору алгоритма:**
- Короткий образец (< 10 символов): BMH
- Длинный образец с повторами: Boyer-Moore
- Гарантированная линейность: KMP (не реализован)
- Малый алфавит (ДНК): KMP предпочтительнее

## API использования

```cpp
#include "search/BoyerMooreSearch.hpp"

// Создание поисковика
BoyerMooreSearch searcher;

// Установка образца (строит таблицы сдвигов)
searcher.setPattern("KOLOKOL");

// Поиск первого вхождения
int pos = searcher.search("KOLKOKOLOKOLL");  // → 3

// Поиск всех вхождений
auto all = searcher.searchAll("KOLOKOL KOLOKOL");  // → [0, 8]

// Статистика
size_t comparisons = searcher.getComparisonCount();

// Переиспользование с другим образцом
searcher.setPattern("ABC");
pos = searcher.search("ABCDEF");  // → 0
```
